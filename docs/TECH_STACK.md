## 2. 技术选型标准

### 2.1 选择原则

1. **前端优先**：聚焦前端技术栈，暂不深入后端/运维领域
2. **实用性**：选择实际项目中常用的技术，而非纯理论
3. **深度优先**：每个技术栈深入讲解原理和底层机制，而非表面 API
4. **体系化**：知识点之间建立关联，形成完整的知识网络

### 2.2 文档质量标准

每个技术栈文档必须包含：

| 要素 | 说明 |
|------|------|
| 概念定义 | 解释"是什么"和"为什么" |
| 工作原理 | 说明内部机制和运作流程 |
| 源码解析 | 关键特性的底层实现 |
| 代码示例 | 配合文字说明的完整代码 |
| 常见误区 | 开发者容易误解的地方 |
| 最佳实践 | 实际开发中的推荐使用方式 |

## 5. 技能栈（Skill System）

### 5.1 核心 Skill

| Skill | 用途 | 状态 |
|-------|------|------|
| `/research` | 深度调研任意主题并生成结构化知识文档 | ✅ |
| `/skill-creator` | 通过问答引导创建新 Skill | ✅ |
| `/study` | 基于费曼学习法的互动教学 | ✅ |
| `/skill-adapter` | Skill 迁移适配向导 | ✅ |
| `/project-start` | 文档优先开发系统初始化 | ✅ |
| `/project-migration` | 传统项目迁移至文档优先系统 | ✅ |
| `/requirement-change` | 项目需求变更处理 | ✅ |
| `/auto-skill` | 检测重复命令模式并建议创建 Skill | ✅ |
| `/kb-trigger` | 检测技术栈缺失并自动调研 | ✅ |

### 5.2 Skill 系统学习路径

```
1. 理解 Skill 概念 → 阅读 SKILL 核心知识体系
       ↓
2. 使用现有 Skill → 实践 /research, /study 等
       ↓
3. 创建新 Skill → 使用 /skill-creator
       ↓
4. 迁移/适配 Skill → 使用 /skill-adapter
```
