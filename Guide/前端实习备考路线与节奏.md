# 前端实习备考路线与节奏

> 目标：拿到满意的前端实习 offer
> 起点：有大厂面试经验，简历过关，AI 协同开发能力强，算法/八股深度需补强
> 当前时间：2026 年 4 月中旬

---

## 整体时间线（月度级别）

| 阶段 | 时间 | 主题 | 核心目标 | 成功标志 |
|------|------|------|---------|---------|
| **M1** | 4月中旬 - 5月中旬 | 面试冲刺期 | 拿到暑期实习 offer | 每周 2-3 场面试，能流畅回答 50 道核心八股，热题 100 能做 70% |
| **M2** | 5月中旬 - 6月中旬 | 秋招预备期 | 提前进组 or 调整策略 | 有一个完整的 AI 前端项目可展示 |
| **M3** | 6月中旬 - 7月底 | 秋招提前批 | 锁定秋招正式批 offer | 3+ 个 offer 保底 |
| **M4** | 8月以后 | 正式批 / 转正 | 最终锁定 offer | 9 月前拿到至少 1 个正式 offer |

### 各阶段能力目标

```
算法:
  M1: 基础 → 中等（热题 100 掌握 70%）
  M2: 保持手感（每天 1-2 题维持熟练度）
  M3: 综合熟练（能独立做大部分 Medium）
  M4: 保持手感（面试前突击复习即可）

八股:
  M1: 单点 → 体系（建立完整知识图谱）
  M2: 场景化（能结合实际项目回答）
  M3: 即兴发挥（被问到任何点都能讲 3 分钟）
  M4: 够用就行（查漏补缺，不再系统学）

项目:
  M1: 迭代功能（RAG 向量化、性能优化）
  M2: 完整产品（AI Agent 全栈应用）
  M3: 开源推广（技术博客、开源项目）
  M4: 实战经验（实习中的实际项目）

投递:
  M1: 高频（每周 10-15 家，多渠道）
  M2: 维持（每周 5-8 家，精准投递）
  M3: 全量（秋招提前批全开，广撒网）
  M4: 收尾（对比 offer，做最终选择）
```

## 每周节奏（M1 面试冲刺期为例）

### 4 周一循环

| 周 | 主题 | 算法重点 | 八股重点 | 项目重点 | 投递节奏 |
|----|------|---------|---------|---------|---------|
| **W1** | JS 基础 | 数组 / 字符串 / 双指针 | JS 核心（闭包 / 原型链 / this / 事件循环） | Knowledge Base RAG 模块 | 周一集中投 |
| **W2** | 浏览器 + React | 链表 / 树 / 遍历 | 渲染流程 / Diff / Fiber / Hooks 源码 | 性能优化专项（Lighthouse 提升） | 周一投 + 周三检查 |
| **W3** | 网络 + 工程化 | 排序 / 二分 / 哈希 | HTTP / 缓存 / 跨域 / Webpack / Vite | AI Agent 功能集成 | 周一投 + 周三检查 |
| **W4** | 模拟周 | 综合刷题（错题重做） | 全主题串讲 | 技术博客输出 | 复盘投递数据 |

### 每周结构模板

```
周一（投递日）：
├── 上午：算法（新题 2 道 + 复习 1 道）
├── 上午：八股（按本周主题深挖）
├── 下午：项目开发
├── 下午：投递（45min，批量投 10-15 家）
└── 晚上：复习 + 查漏补缺

周二 - 周四（深度学习日）：
├── 上午：算法（新题 1-2 道 + 复习前天的题）
├── 上午：八股（按计划推进主题）
├── 下午：项目开发（深度开发，不中断）
├── 下午：旧知识复习（30min）
└── 晚上：新知识 / AI 相关（1.5h）

周五（复盘日）：
├── 上午：算法（本周错题重做）
├── 上午：八股（本周主题串讲 + 录音自测）
├── 下午：投递检查（45min）+ 项目继续
├── 下午：复习（30min）
└── 晚上：周总结（写本周学到了什么、下周怎么调）

周六（模拟面试日）：
├── 上午：完整模拟面试 1 次（1.5-2h）
├── 下午：复盘模拟面试（找薄弱点，记录追踪表）
└── 晚上：轻松，看技术博客 / 开源项目

周日（调整日）：
├── 上午：补本周遗留问题（1-2h）
├── 下午：休息
└── 晚上：规划下周（30min 看计划表）
```

## 各部分具体执行方法

### 算法（1.5h / 天）

```
前 45min：精做 1 题
├── 读题 → 想 10 分钟，想不出看题解
├── 不看代码，只看思路，自己实现
├── 写完后对比最优解，找差距
└── 记录：这道题的"模式"是什么？（滑动窗口？双指针？单调栈？）

后 45min：复习 + 二刷
├── 重做前天的题（不看题解）
├── 做 1 道简单题热身
└── 标记"卡壳点"，写一句话总结

每周六：本周错题重做
每月末：综合模拟（限时 3 题，90 分钟）
```

**推荐顺序**：灵神题单基础部分 → LeetCode 热题 100 → 剑指 Offer

**目标**：掌握 20 个算法模式，覆盖 90% 面试算法题

### 八股文（1.25h / 天）

```
第 1 阶段（20min）：概念梳理
└── 用自己的话写一遍"是什么 + 为什么 + 怎么用"

第 2 阶段（30min）：底层原理
├── 看源码片段 / 技术博客
├── 搞懂内部机制
└── 画一张流程图 / 关系图

第 3 阶段（20min）：面试题对照
├── 这个主题在面试中会被怎么问？
├── 列出 3-5 个常见问法
└── 每个问题写一个"口头回答草稿"

第 4 阶段（15min）：录音自测
└── 大声回答，录下来听，看是否流畅、有深度
```

**6 周主题排期**：

| 周 | 主题 | 核心问题 |
|----|------|---------|
| 1 | JS 基础 | 闭包、原型链、this、事件循环 |
| 2 | 浏览器 | 渲染流程、缓存、网络、安全 |
| 3 | React | Hooks 原理、Fiber、Diff、状态管理 |
| 4 | 工程化 | Webpack/Vite、性能优化、TypeScript |
| 5 | 网络 | HTTP/1.1→3、WebSocket、跨域 |
| 6 | Node.js + 框架 | SSR、BFF、数据库基础 |

### 项目 / 实践（2h / 天）

```
以 Knowledge Base 项目为载体，每周聚焦一个技术深度点：

Week 1：实现 RAG 问答系统
├── 把知识库向量化
├── 接 LLM API 做智能问答
└── 面试可以说："我做了一个基于 RAG 的知识问答系统"

Week 2：性能优化
├── Lighthouse 测分
├── 代码分割、懒加载、缓存策略
└── 面试可以说："我把首屏从 3s 优化到 1.2s"

Week 3：工程化
├── CI/CD 流程、ESLint/Prettier、单元测试
└── 面试可以说："我搭建了一套完整的前端工程化体系"

Week 4：Agent 功能
├── 集成 MCP 协议
├── 实现一个简单的学习 Agent
└── 面试可以说："我做了一个 AI Agent 应用"

每个迭代结束写一篇技术博客，发到掘金 / 知乎。
```

### 旧知识巩固（30min / 天）

```
间隔重复策略：
├── 1 天前学的（短期记忆强化）
├── 3 天前学的（防止遗忘）
├── 7 天前学的（长期记忆巩固）
└── 标记过"不会"的题目（重点突破）

关键动作：
├── 不看笔记，自己回忆
├── 回忆不出来的，标记为重难点
└── 能流畅说出来的，标记为已通过
```

### 投递管理

```
每周节奏：
├── 周一：集中投递 10-15 家（30-45min）
├── 周三：检查进度，补投 3-5 家
└── 周五：复盘（记录投递结果，更新追踪表）

投递渠道优先级：
├── 第一梯队（回复率高）：内推、Boss 直聘、公司官网
├── 第二梯队（量大但慢）：牛客校招、实习僧、拉勾
└── 第三梯队（随缘）：朋友圈、公众号、微信群

投递追踪表（Excel / Notion）：
| 公司 | 岗位 | 投递日期 | 渠道 | 状态 | 面试轮次 | 反馈 | 备注 |
```

### 简历更新

```
触发条件（满足任一即更新）：
├── 完成了新的项目迭代 → 立即更新项目描述
├── 拿到了新的证书 / 比赛结果 → 立即更新
├── 连续 3 次面试被问同一个问题 → 把答案写进简历
├── 面试时发现自己某个亮点没写进简历 → 立即补
└── 每两周固定检查一次

每次面试后 10min 复盘：
├── 面试官问了什么我没写在简历上的？→ 亮点则加进简历
├── 面试官质疑了什么我写了的内容？→ 描述不清则改精准
├── 面试官对我哪个项目最感兴趣？→ 把那个项目往上调
└── 哪个环节我表现不好？→ 面试表达问题，不改简历
```

## 动态调整机制

根据反馈调整策略：

### 连续 2 周没有面试

```
检查：
├── 投递量够吗？（每周 <10 家 → 加量）
├── 渠道对吗？（纯网申 → 加内推 + Boss 直聘）
├── 简历需要改吗？
└── 行动：找学长学姐要内推，Boss 上主动聊 HR
```

### 面试连续挂一面

```
检查：
├── 八股深度不够 → 加深原理层学习
├── 算法做不出来 → 降低刷题范围，先保 Medium
├── 表达有问题 → 增加录音自测频率
└── 行动：每次面试后 30min 内写面经，立刻复盘
```

### 一面过但二面挂

```
检查：
├── 场景设计能力不足 → 多看大厂技术博客
├── 项目深度不够 → 把现有项目做更深
└── 行动：针对二面高频题（架构设计、性能优化）专项练习
```

### 拿到 offer

```
├── 不急着接受，继续面更好的
├── 提前进组实习，积累实际经验
└── 目标从"拿 offer"变成"争取转正"
```

## 核心原则

```
每天学，每周验，每月调。

算法：精做不贪多
八股：深挖不背诵
项目：做深度不做广度
复习：间隔重复
查漏：定向补弱，不漫无目的
投递：批量化，不每天刷
简历：根据反馈微调，不大改

最重要的检验标准：
每周能回答——我这周比上周强在哪？
```
