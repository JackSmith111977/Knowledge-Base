# 虚拟列表核心知识体系

> 虚拟列表 (Virtual List) 是一种只渲染可视区域内数据项的性能优化技术，通过动态计算可视区域、按需渲染 DOM、占位元素模拟完整高度，将内存占用从 O(n) 降至 O(1)，实现万级数据流畅滚动。

**特色：** 深入原理 + 多框架实现 + 主流库对比 + 实战案例

## 1. 概述

### 1.1 什么是虚拟列表

**虚拟列表 (Virtual List)**，也称为"窗口化列表"(Windowed List)，是一种优化长列表渲染性能的前端技术。

**核心思想：** 只渲染当前视口 (Viewport) 内可见的元素，对于未出现在视口中的元素进行"虚拟化"处理——即不占用真实的 DOM 节点，而是用占位元素撑开滚动高度。

### 1.2 为什么需要虚拟列表

当页面渲染一个很长的列表时 (如上万条数据)，传统的渲染方式会一次性创建所有 DOM 节点：

```html
<!-- 传统渲染：创建 10000 个 li 元素 -->
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <!-- ... 9998 个 li -->
  <li>Item 10000</li>
</ul>
```

**这会导致三大性能问题：**

| 问题 | 描述 | 影响 |
|------|------|------|
| 页面加载慢 | 需要创建大量 DOM 节点 | 首屏时间飙升 |
| 内存占用大 | 大量 DOM 节点占用内存 | 可能导致内存泄漏 |
| 滚动卡顿 | 浏览器需要维护和渲染大量节点 | 帧率下降，用户体验差 |

**实际案例：** 2025 年 Q4 某头部电商平台技术复盘显示，其订单列表页因数据量突破 800 万行，采用传统分页方案导致首屏加载时间飙升至 12.3 秒，直接引发 17% 的用户流失。

### 1.3 虚拟列表的价值

虚拟列表通过"可视区域渲染 + 动态占位"机制，带来以下价值：

| 优化指标 | 传统列表 | 虚拟列表 | 提升幅度 |
|----------|----------|----------|----------|
| DOM 节点数 | O(n) | O(1) | 减少 90% 以上 |
| 内存占用 | 全量 | 仅可视区域 | 降低 50%-80% |
| 滚动帧率 | 10-30fps | 55-60fps | 提升 2-3 倍 |
| 首屏加载 | 全量渲染 | 仅渲染可见 | 提速 5-10 倍 |

**某金融科技公司实战案例：** 其风控报表系统在 2026 年 2 月升级虚拟滚动后，DOM 节点数从 42 万骤降至 500 个，内存泄漏率下降 92%，滚动帧率稳定在 58fps 以上。

### 1.4 应用场景

虚拟列表适用于以下典型场景：

| 场景 | 描述 | 典型案例 |
|------|------|----------|
| 大数据表格 | ERP、CRM 系统展示成千上万条数据记录 | 电商订单列表 |
| 消息聊天界面 | 长对话历史或群聊消息，需快速滚动浏览 | IM 聊天记录 |
| 无限滚动 feed 流 | 社交媒体动态、新闻流 | 微博、抖音 feed |
| 下拉选择器 | 超长选项列表的选择组件 | 省市区选择器 |
| 日志查看器 | 大量日志条目的展示 | 服务器日志 |

## 2. 核心概念

### 2.1 可视区域 (Viewport)

**定义：** 用户当前能够看到的列表区域，即容器内可见的部分。

**计算方式：**
- 容器高度 (`clientHeight` 或 `offsetHeight`)
- 滚动位置 (`scrollTop`)
- 可视区域起始位置 = `scrollTop`
- 可视区域结束位置 = `scrollTop + clientHeight`

### 2.2 缓冲区 (Overscan / Buffer)

**定义：** 在可视区域上下额外渲染的若干条目，用于防止快速滚动时出现空白闪烁。

**作用：**
- 防止用户快速滚动时看到空白区域
- 提前渲染即将进入可视区域的内容
- 提供更流畅的滚动体验

**缓冲区大小计算：**
```javascript
// 通常设置为可视区域可容纳 item 数的 1-2 倍
const overscan = Math.ceil(viewportHeight / itemHeight) * 1.5
```

### 2.3 DOM 复用 (Recycling)

**定义：** 当列表项滚出可视区域时，不销毁其 DOM 节点，而是复用于渲染新进入可视区域的项。

**实现方式：**
1. 维护一个 DOM 节点池
2. 当 item 滚出可视区时，回收到池中
3. 当需要渲染新 item 时，从池中取出复用的 DOM

**优势：** 减少 DOM 创建和销毁开销，提升滚动性能。

### 2.4 占位元素 (Placeholder)

**定义：** 用于撑开容器滚动高度的虚拟元素，代替未渲染的真实数据。

**实现方式：**
- **方式一：** 使用 `padding` 或 `margin` 撑开高度
- **方式二：** 使用一个空白 `div`，高度 = 总数据量 × 单项高度
- **方式三：** 使用 `transform` 偏移渲染区域

```html
<!-- 占位容器示例 -->
<div class="virtual-list" style="height: 500px; overflow: auto;">
  <!-- 占位元素：撑开滚动高度 -->
  <div style="height: 500000px; position: relative;">
    <!-- 实际渲染的可视区域 -->
    <div style="transform: translateY(1000px);">
      <div>Item 21</div>
      <div>Item 22</div>
      <!-- ... 仅渲染可见的 20 项 -->
    </div>
  </div>
</div>
```

### 2.5 渲染窗口 (Render Window)

**定义：** 实际渲染的 DOM 区域，包含可视区域 + 上下缓冲区。

**渲染窗口计算：**
```javascript
const startIndex = max(0, floor(scrollTop / itemHeight) - overscan)
const endIndex = min(
  totalItems,
  ceil((scrollTop + viewportHeight) / itemHeight) + overscan
)
```

## 3. 工作原理

### 3.1 核心流程图

```
┌─────────────────────────────────────────────────────────────┐
│                      用户滚动列表                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  1. 监听 scroll 事件，获取 scrollTop                           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 计算可视区域范围                                          │
│     startIndex = floor(scrollTop / itemHeight)                │
│     endIndex = startIndex + ceil(viewportHeight / itemHeight) │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 计算缓冲区范围                                            │
│     renderStart = max(0, startIndex - overscan)               │
│     renderEnd = min(totalItems, endIndex + overscan)          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 截取数据，渲染可视区域 + 缓冲区                              │
│     visibleItems = data.slice(renderStart, renderEnd)         │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  5. 计算偏移量，使用 transform/absolute 定位到正确位置          │
│     offsetY = renderStart * itemHeight                        │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 关键计算步骤

#### 3.2.1 可视区域计算

```javascript
// 已知条件
const scrollTop = container.scrollTop;        // 滚动位置
const viewportHeight = container.clientHeight; // 可视区域高度
const itemHeight = 50;                         // 单项高度
const totalItems = 10000;                      // 总数据量

// 计算可视区域索引范围
const visibleStartIndex = Math.floor(scrollTop / itemHeight);
const visibleEndIndex = Math.ceil(
  (scrollTop + viewportHeight) / itemHeight
);
```

#### 3.2.2 缓冲区计算

```javascript
// 缓冲区大小：可视区域可容纳 item 数的 1 倍
const overscanCount = Math.ceil(viewportHeight / itemHeight);

// 渲染范围 (包含缓冲区)
const renderStartIndex = Math.max(0, visibleStartIndex - overscanCount);
const renderEndIndex = Math.min(
  totalItems,
  visibleEndIndex + overscanCount
);
```

#### 3.2.3 偏移量计算

```javascript
// 计算偏移量，让渲染的项显示在正确位置
const offsetY = renderStartIndex * itemHeight;

// 应用偏移 (方式一：transform)
renderContainer.style.transform = `translateY(${offsetY}px)`;

// 应用偏移 (方式二：absolute + top)
renderContainer.style.top = `${offsetY}px`;
```

### 3.3 滚动事件监听优化

直接监听 `scroll` 事件会导致高频触发，需要进行节流/防抖优化：

```javascript
// 方式一：使用 throttle 节流 (推荐)
import throttle from 'lodash/throttle';

const handleScroll = throttle((e) => {
  const scrollTop = e.target.scrollTop;
  // ... 更新渲染逻辑
}, 16); // 约 60fps

container.addEventListener('scroll', handleScroll);

// 方式二：使用 requestAnimationFrame
let ticking = false;

const handleScroll = (e) => {
  if (!ticking) {
    window.requestAnimationFrame(() => {
      const scrollTop = e.target.scrollTop;
      // ... 更新渲染逻辑
      ticking = false;
    });
    ticking = true;
  }
};

// 方式三：使用 Intersection Observer (最新方案)
// 适用于复杂场景，可检测元素是否进入可视区域
```

### 3.4 位置缓存策略 (动态高度场景)

对于高度不固定的列表项，需要缓存每个 item 的位置信息：

```javascript
// 位置缓存对象
const positionCache = new Map();

// 获取 item 高度 (优先从缓存读取)
const getItemHeight = (index) => {
  if (positionCache.has(index)) {
    return positionCache.get(index);
  }
  // 测量实际高度
  const height = measureItem(index);
  positionCache.set(index, height);
  return height;
};

// 使用二分查找快速定位滚动位置对应的索引
const findIndexByOffset = (offset) => {
  let low = 0, high = totalItems - 1;
  while (low <= high) {
    const mid = Math.floor((low + high) / 2);
    const midOffset = getPositionOffset(mid);
    if (midOffset < offset) {
      low = mid + 1;
    } else if (midOffset > offset) {
      high = mid - 1;
    } else {
      return mid;
    }
  }
  return low;
};
```

## 4. 基础实现

### 4.1 React 基础实现 (固定高度)

```jsx
import React, { useState, useRef, useEffect, useMemo } from 'react';

function VirtualList({
  items,           // 所有数据
  itemHeight = 50, // 固定高度
  height = 400,    // 容器高度
  overscan = 5     // 缓冲区大小
}) {
  const containerRef = useRef(null);
  const [scrollTop, setScrollTop] = useState(0);

  // 计算总高度
  const totalHeight = useMemo(() => items.length * itemHeight, [items, itemHeight]);

  // 计算可视区域索引范围
  const visibleCount = Math.ceil(height / itemHeight);
  const startIndex = Math.floor(scrollTop / itemHeight);
  const endIndex = Math.min(items.length, startIndex + visibleCount);

  // 计算缓冲区范围
  const renderStart = Math.max(0, startIndex - overscan);
  const renderEnd = Math.min(items.length, endIndex + overscan);

  // 截取可视数据
  const visibleItems = useMemo(() =>
    items.slice(renderStart, renderEnd),
    [items, renderStart, renderEnd]
  );

  // 计算偏移量
  const offsetY = renderStart * itemHeight;

  // 滚动处理
  const handleScroll = (e) => {
    setScrollTop(e.target.scrollTop);
  };

  return (
    <div
      ref={containerRef}
      style={{
        height: `${height}px`,
        overflow: 'auto',
        position: 'relative'
      }}
      onScroll={handleScroll}
    >
      {/* 占位元素：撑开滚动高度 */}
      <div style={{ height: `${totalHeight}px`, position: 'relative' }}>
        {/* 实际渲染区域 */}
        <div
          style={{
            position: 'absolute',
            top: 0,
            transform: `translateY(${offsetY}px)`,
            width: '100%'
          }}
        >
          {visibleItems.map((item, index) => (
            <div
              key={item.id}
              style={{
                height: `${itemHeight}px`,
                display: 'flex',
                alignItems: 'center',
                borderBottom: '1px solid #eee'
              }}
            >
              {item.content}
            </div>
          ))}
        </div>
      </div>
    </div>
  );
}

export default VirtualList;
```

### 4.2 Vue 3 基础实现 (固定高度)

```vue
<script setup lang="ts">
import { ref, computed, onMounted } from 'vue'

interface Props {
  items: any[]
  itemHeight?: number
  height?: number
  buffer?: number
}

const props = withDefaults(defineProps<Props>(), {
  itemHeight: 40,
  height: 300,
  buffer: 5
})

// 滚动容器 DOM
const containerRef = ref<HTMLDivElement>()
// 当前滚动位置
const scrollTop = ref(0)

// 总高度 (撑出滚动条)
const totalHeight = computed(() =>
  props.items.length * props.itemHeight
)

// 可视区域计算
const visibleStartIndex = computed(() =>
  Math.floor(scrollTop.value / props.itemHeight)
)

const visibleEndIndex = computed(() =>
  Math.min(
    props.items.length,
    Math.ceil((scrollTop.value + props.height) / props.itemHeight)
  )
)

// 包含缓冲区
const renderStartIndex = computed(() =>
  Math.max(0, visibleStartIndex.value - props.buffer)
)

const renderEndIndex = computed(() =>
  Math.min(props.items.length, visibleEndIndex.value + props.buffer)
)

// 可视数据
const visibleItems = computed(() =>
  props.items.slice(renderStartIndex.value, renderEndIndex.value)
)

// 偏移量
const offsetY = computed(() =>
  renderStartIndex.value * props.itemHeight
)

// 滚动处理
const handleScroll = (e: Event) => {
  scrollTop.value = (e.target as HTMLDivElement).scrollTop
}
</script>

<template>
  <div
    ref="containerRef"
    :style="{
      height: `${props.height}px`,
      overflow: 'auto',
      position: 'relative'
    }"
    @scroll="handleScroll"
  >
    <!-- 占位容器 -->
    <div :style="{ height: `${totalHeight}px`, position: 'relative' }">
      <!-- 渲染区域 -->
      <div
        :style="{
          position: 'absolute',
          top: 0,
          transform: `translateY(${offsetY}px)`,
          width: '100%'
        }"
      >
        <div
          v-for="item in visibleItems"
          :key="item.id"
          :style="{
            height: `${props.itemHeight}px`,
            display: 'flex',
            alignItems: 'center',
            borderBottom: '1px solid #eee'
          }"
        >
          {{ item.content }}
        </div>
      </div>
    </div>
  </div>
</template>
```

### 4.3 关键实现要点总结

| 要点 | 实现方式 | 注意事项 |
|------|----------|----------|
| 固定高度 | `itemHeight` 参数 | 确保所有项高度一致 |
| 滚动容器 | `overflow: auto` + 固定高度 | 必须设置容器高度 |
| 占位元素 | `height: totalHeight` | 撑开完整滚动高度 |
| 偏移定位 | `transform: translateY` | 性能优于 `top` |
| 滚动监听 | `@scroll` / `onScroll` | 需节流优化 |
| 数据截取 | `slice(start, end)` | 包含缓冲区 |

## 5. 高级特性

### 5.1 动态高度支持

当列表项高度不固定时，需要额外处理：

**实现思路：**
1. 首次渲染时测量每个 item 的实际高度
2. 缓存高度到 Map 或数组
3. 使用累加高度计算位置偏移
4. 使用二分查找快速定位索引

```javascript
// 动态高度虚拟列表核心逻辑
class DynamicVirtualList {
  constructor(data, options = {}) {
    this.data = data;
    this.itemHeights = new Map(); // 高度缓存
    this.itemOffsets = new Map(); // 偏移缓存
    this.estimatedItemHeight = options.estimatedHeight || 50;
  }

  // 获取 item 高度
  getItemHeight(index) {
    if (this.itemHeights.has(index)) {
      return this.itemHeights.get(index);
    }
    return this.estimatedItemHeight;
  }

  // 获取 item 偏移量
  getItemOffset(index) {
    if (this.itemOffsets.has(index)) {
      return this.itemOffsets.get(index);
    }
    let offset = 0;
    for (let i = 0; i < index; i++) {
      offset += this.getItemHeight(i);
    }
    return offset;
  }

  // 获取总高度
  getTotalHeight() {
    return this.getItemOffset(this.data.length);
  }

  // 查找索引 (二分查找)
  findIndexByOffset(offset) {
    let low = 0, high = this.data.length - 1;
    while (low <= high) {
      const mid = Math.floor((low + high) / 2);
      const midOffset = this.getItemOffset(mid);
      if (midOffset < offset) {
        low = mid + 1;
      } else if (midOffset > offset) {
        high = mid - 1;
      } else {
        return mid;
      }
    }
    return low;
  }

  // 测量并缓存高度
  measureItem(index, element) {
    const height = element.offsetHeight;
    this.itemHeights.set(index, height);
    // 更新后续 item 的偏移缓存
    this.invalidateOffsetCache(index);
  }
}
```

### 5.2 横向虚拟列表

虚拟列表也可应用于横向滚动场景：

```jsx
function HorizontalVirtualList({ items, itemWidth = 100, width = 600 }) {
  const [scrollLeft, setScrollLeft] = useState(0);

  const visibleCount = Math.ceil(width / itemWidth);
  const startIndex = Math.floor(scrollLeft / itemWidth);
  const endIndex = Math.min(items.length, startIndex + visibleCount);
  const overscan = 3;

  const renderStart = Math.max(0, startIndex - overscan);
  const renderEnd = Math.min(items.length, endIndex + overscan);
  const visibleItems = items.slice(renderStart, renderEnd);
  const offsetX = renderStart * itemWidth;

  return (
    <div
      style={{
        width: `${width}px`,
        overflow: 'auto',
        whiteSpace: 'nowrap'
      }}
      onScroll={(e) => setScrollLeft(e.target.scrollLeft)}
    >
      <div style={{
        width: `${items.length * itemWidth}px`,
        height: '100%',
        position: 'relative'
      }}>
        <div style={{
          position: 'absolute',
          left: 0,
          transform: `translateX(${offsetX}px)`,
          display: 'flex'
        }}>
          {visibleItems.map(item => (
            <div
              key={item.id}
              style={{ width: `${itemWidth}px` }}
            >
              {item.content}
            </div>
          ))}
        </div>
      </div>
    </div>
  );
}
```

### 5.3 预加载与懒加载

对于需要异步加载数据的场景：

```javascript
// 预加载策略
class VirtualListWithPreload {
  constructor(options) {
    this.data = options.data;
    this.loadData = options.loadData; // 异步加载函数
    this.preloadThreshold = options.preloadThreshold || 5;
  }

  // 检测是否需要预加载
  checkPreload(visibleEndIndex) {
    const remainingItems = this.data.length - visibleEndIndex;
    if (remainingItems <= this.preloadThreshold) {
      // 触发预加载
      this.preloadMore();
    }
  }

  // 加载更多数据
  async preloadMore() {
    if (this.loading) return;
    this.loading = true;
    try {
      const newData = await this.loadData();
      this.data = [...this.data, ...newData];
    } finally {
      this.loading = false;
    }
  }
}
```

function VirtualListWithScrollTo({ items, itemHeight, scrollToIndex }) {

        behavior: 'smooth'
      });
    }
  }, [scrollToIndex, itemHeight]);

  return (
    <div ref={containerRef} /* ... */>
      {/* ... */}
    </div>
  );
}

## 6. 主流库对比

### 6.1 React 生态

| 库 | 特点 | 适用场景 | Bundle Size |
|----|------|----------|-------------|
| **react-window** | 轻量级，API 简洁 | 固定高度、简单动态高度 | ~12KB |

### 7.2 无限滚动表格

```jsx
function InfiniteScrollTable({ columns, fetchApi }) {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(false);
  const [hasMore, setHasMore] = useState(true);
  const containerRef = useRef();

  const loaderRef = useRef();

  // 使用 Intersection Observer 实现无限滚动
  useEffect(() => {
    const observer = new IntersectionObserver(
      (entries) => {
        if (entries[0].isIntersecting && !loading && hasMore) {
          loadMore();
        }
      },
      { threshold: 0.1 }
    );

    if (loaderRef.current) {
      observer.observe(loaderRef.current);
    }

    return () => observer.disconnect();
  }, [loading, hasMore]);

  const loadMore = async () => {
    setLoading(true);
    try {
      const newData = await fetchApi({ offset: data.length, limit: 50 });
      setData(prev => [...prev, ...newData]);
      setHasMore(newData.length === 50);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div ref={containerRef} className="table-container">
      <table>
        <thead>
          <tr>
            {columns.map(col => <th key={col.key}>{col.title}</th>)}
          </tr>
        </thead>
        <tbody>
          {data.map(row => (
            <tr key={row.id}>
              {columns.map(col => <td key={col.key}>{row[col.key]}</td>)}
            </tr>
          ))}
        </tbody>
      </table>
      <div ref={loaderRef} className="loading-trigger">
        {loading && <span>加载中...</span>}
        {!hasMore && <span>没有更多数据了</span>}
      </div>
    </div>
  );
}
```

function LazyImage({ src, alt }) {
  const imgRef = useRef();
  const [isVisible, setIsVisible] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsVisible(true);
          observer.disconnect();
        }
      },
      { rootMargin: '100px' }
    );

    if (imgRef.current) {
      observer.observe(imgRef.current);
    }

    return () => observer.disconnect();
  }, []);

      {isVisible ? (
        <img src={src} alt={alt} loading="lazy" />
      ) : (
        <div className="image-placeholder">加载中...</div>
      )}
    </div>
  );
}

## 8. 常见问题

### 8.1 滚动抖动/跳动

**问题表现：** 滚动时列表内容突然跳动，无法平滑滚动。

**常见原因：**
1. 动态高度计算不准确
2. 异步加载数据导致高度变化
3. 图片加载完成后改变项高度

**解决方案：**

```javascript
// 1. 使用精确的高度缓存
const heightCache = useRef(new Map());

const getEstimatedOffset = (index) => {
  let offset = 0;
  for (let i = 0; i < index; i++) {
    offset += heightCache.current.get(i) || estimatedHeight;
  }
  return offset;
};

// 2. 高度更新后重新计算位置
const updateHeight = (index, newHeight) => {
  const oldHeight = heightCache.current.get(index) || estimatedHeight;
  heightCache.current.set(index, newHeight);

  // 计算高度差，调整滚动位置
  const heightDiff = newHeight - oldHeight;
  if (index < currentVisibleStartIndex) {
    containerRef.current.scrollTop += heightDiff;
  }
};

// 3. 使用 ResizeObserver 监听高度变化
useEffect(() => {
  const resizeObserver = new ResizeObserver((entries) => {
    for (const entry of entries) {
      const index = Number(entry.target.dataset.index);
      updateHeight(index, entry.contentRect.height);
    }
  });

  // 观察所有渲染的项
  renderedItems.forEach(item => {
    const el = document.querySelector(`[data-index="${item.index}"]`);
    if (el) resizeObserver.observe(el);
  });

  return () => resizeObserver.disconnect();
}, [renderedItems]);
```

### 8.2 快速滚动时出现空白

**问题表现：** 快速滚动时，列表底部或顶部出现空白区域。

**解决方案：**
1. 增加缓冲区大小 (`overscan`)
2. 使用 `IntersectionObserver` 替代 `scroll` 监听
3. 预加载下一批数据

```javascript
// 增加缓冲区
const virtualizer = useVirtualizer({
  count: items.length,
  getScrollElement: () => parentRef.current,
  estimateSize: () => 50,
  overscan: 10, // 增加缓冲区域
  initialRect: { height: 600 }, // 初始可视区域
});

// 使用 IntersectionObserver
useEffect(() => {
  const observer = new IntersectionObserver(
    (entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          // 提前加载
          preloadMore();
        }
      });
    },
    { rootMargin: '500px' } // 提前 500px 加载
  );
}, []);
```

### 8.3 内存泄漏问题

**问题表现：** 长时间使用后内存持续增长，页面变卡。

**常见原因：**
1. 事件监听器未清理
2. 闭包引用导致 DOM 无法释放
3. 缓存数据无限增长

**解决方案：**

```javascript
// 1. 清理事件监听
useEffect(() => {
  const container = containerRef.current;
  const handleScroll = throttle(() => { /* ... */ }, 16);

  container.addEventListener('scroll', handleScroll);
  return () => container.removeEventListener('scroll', handleScroll);
}, []);

// 2. 限制缓存大小
const MAX_CACHE_SIZE = 1000;

const cacheHeight = (index, height) => {
  if (heightCache.size > MAX_CACHE_SIZE) {
    // 清除最早缓存的项
    const firstKey = heightCache.keys().next().value;
    heightCache.delete(firstKey);
  }
  heightCache.set(index, height);
};

// 3. 使用 WeakMap 存储 DOM 引用
const domRefs = new WeakMap();
```

### 8.4 动态数据更新导致闪烁

**问题表现：** 数据更新时列表闪烁或滚动位置丢失。

**解决方案：**

```javascript
// 1. 使用稳定的 key
{virtualItems.map(item => (
  <div key={data[item.index].id} /* ... */ />
  // 不要使用 index 作为 key
))}

// 2. 保存和恢复滚动位置
const [scrollPosition, setScrollPosition] = useState(0);

const updateData = async (newData) => {
  const currentScroll = containerRef.current.scrollTop;
  setData(newData);
  // 等待下一帧恢复滚动位置
  requestAnimationFrame(() => {
    containerRef.current.scrollTop = currentScroll;
  });
};

// 3. 使用 useMemo 缓存计算结果
const visibleItems = useMemo(() => {
  return data.slice(startIndex, endIndex);
}, [data, startIndex, endIndex]);
```
