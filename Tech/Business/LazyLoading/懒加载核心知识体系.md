# 懒加载核心知识体系

> 懒加载 (Lazy Loading) 是一种按需加载的性能优化技术，通过延迟加载非关键资源直到用户真正需要时才加载，显著减少初始加载体积、提升首屏加载速度、节省带宽消耗。

**特色：** 核心原理 + 多框架实现 + 实战案例 + 性能优化

## 1. 概述

### 1.1 什么是懒加载

**懒加载 (Lazy Loading)**，也称为惰性加载，是一种延迟加载非关键资源以提高性能的技术。

**核心思想：** 只在资源即将进入视口 (Viewport) 或用户真正需要时才加载，而非页面初始化时全部加载。

**对比传统加载方式：**

| 加载方式 | 描述 | 缺点 |
|----------|------|------|
| **即时加载 (Eager Loading)** | 页面初始化时加载所有资源 | 首屏加载慢、带宽浪费 |
| **懒加载 (Lazy Loading)** | 按需加载，进入视口才加载 | 快速滚动时可能延迟 |

### 1.2 为什么需要懒加载

**背景问题：** 现代前端应用 (尤其是 SPA) 往往打包出巨大的 Bundle 文件，导致：
- 首屏加载时间长达 10 秒以上
- 用户流失率高达 17%
- 带宽浪费严重

**实际案例：** 某电商项目单个 bundle.js 高达 5MB，通过懒加载优化后：
- 初始 bundle 减少到 500KB
- 首屏加载时间从 10 秒减少到 4 秒
- 首屏加载体积减少 50%-80%

### 1.3 懒加载的价值

| 优化指标 | 优化前 | 优化后 | 提升幅度 |
|----------|--------|--------|----------|
| 初始加载体积 | 5MB | 500KB | 减少 90% |
| 首屏加载时间 | 10 秒 | 4 秒 | 提速 60% |
| 初始请求数 | 全量 | 按需 | 减少 70% |
| 内存占用 | 全量 | 按需 | 降低 50%-80% |

### 1.4 应用场景

懒加载适用于以下典型场景：

| 场景 | 描述 | 典型案例 |
|------|------|----------|
| **图片懒加载** | 长列表中非首屏图片延迟加载 | 电商商品列表、社交媒体 feed |
| **路由懒加载** | 访问路由时才加载对应组件 | SPA 应用、后台管理系统 |
| **组件懒加载** | 非首屏必需的组件延迟加载 | 模态框、复杂图表、富文本编辑器 |
| **数据懒加载** | 滚动到底部再加载更多数据 | 无限滚动列表、分页加载 |
| **第三方库懒加载** | 大型第三方库按需加载 | echarts、moment、地图 SDK |

## 2. 核心概念

### 2.1 视口 (Viewport)

**定义：** 用户当前能够看到的浏览器可视区域。

**相关属性：**
```javascript
// 视口高度
const viewportHeight = window.innerHeight;

// 视口宽度
const viewportWidth = window.innerWidth;
```

### 2.2 交叉观察器 (Intersection Observer)

**定义：** 浏览器原生 API，异步监听目标元素与祖先容器或视口的交叉状态变化。

**核心优势：**
- 由浏览器底层负责几何计算，显著降低主线程开销
- 合并回调队列，避免频繁触发
- 提供 `rootMargin`、`thresholds` 等灵活调谐参数

**基本用法：**
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      // 元素进入视口，执行懒加载
      entry.target.src = entry.target.dataset.src;
      observer.unobserve(entry.target);
    }
  });
}, {
  root: null,        // 默认视口
  rootMargin: '0px', // 根节点 margin
  threshold: 0.1     // 交叉比例阈值
});

observer.observe(targetElement);
```

**核心 API：**

| 方法 | 说明 |
|------|------|
| `observe()` | 开始监听目标元素 |
| `unobserve()` | 停止监听特定元素 |
| `disconnect()` | 停止监听所有目标 |
| `takeRecords()` | 返回所有观察目标的交叉信息 |

### 2.3 代码分割 (Code Splitting)

**定义：** 将庞大的代码包拆分成多个小的 Chunk，让用户只下载当前页面所需的代码。

**实现方式：**
- **路由级分割：** 为每个路由生成独立的 Chunk
- **组件级分割：** 大型组件单独拆分
- **第三方库分离：** node_modules 单独打包为 vendor chunk

**效果：** 首屏加载体积可减少 50%-80%。

### 2.4 动态导入 (Dynamic Import)

**定义：** 使用 `import()` 语法动态加载模块，返回 Promise。

```javascript
// 动态导入
const module = await import('./module.js');

// React 组件懒加载
const LazyComponent = lazy(() => import('./LazyComponent'));
```

### 2.5 Suspense (异步边界)

**定义：** React 提供的异步边界组件，用于在懒加载组件未准备好时显示 fallback。

```jsx
<Suspense fallback={<Loading />}>
  <LazyComponent />
</Suspense>
```

## 3. 工作原理

### 3.1 懒加载核心流程

```
┌─────────────────────────────────────────────────────────────┐
│  页面初始化：只加载首屏关键资源                                │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  监听视口变化：使用 Intersection Observer 或 scroll 事件       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  检测元素进入视口：计算元素与视口的相对位置                    │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  触发加载：将 data-src 赋值给 src，或执行动态 import()         │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  加载完成：显示真实内容，移除监听                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 可视性检测算法

**传统方案（基于 scroll 事件）：**
```javascript
function checkVisible(element) {
  const rect = element.getBoundingClientRect();
  const viewportHeight = window.innerHeight;

  // 元素顶部进入视口下方 且 底部未完全离开视口
  return rect.top < viewportHeight && rect.bottom > 0;
}

window.addEventListener('scroll', throttle(() => {
  lazyElements.forEach(el => {
    if (checkVisible(el)) {
      el.src = el.dataset.src;
    }
  });
}, 100));
```

**现代方案（Intersection Observer）：**
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.src = entry.target.dataset.src;
      observer.unobserve(entry.target); // 加载完成后停止观察
    }
  });
}, {
  threshold: 0.1 // 10% 可见时触发
});
```

### 3.3 React.lazy 工作机制

**核心流程：**
```
1. 首次渲染 <LazyComponent />
   ↓
2. React.lazy 触发 import()，发起网络请求加载 chunk
   ↓
3. 组件尚未加载完成时，lazy 内部会 throw 一个 Promise
   ↓
4. Suspense 捕获该 Promise，暂停子树渲染，显示 fallback
   ↓
5. 代码加载并解析完成后，React 自动重新渲染真实组件
```

**关键机制：** "抛出 Promise" + "异步边界捕获"

```jsx
// React.lazy 内部简化逻辑
function lazy(getComponent) {
  let Component;
  let promise;

  return function LazyComponent(props) {
    if (!promise) {
      promise = getComponent().then(module => {
        Component = module.default;
      });
    }

    if (Component) {
      return <Component {...props} />;
    }

    throw promise; // 抛出 Promise 让 Suspense 捕获
  };
}
```

### 3.4 浏览器渲染流水线

**懒加载对渲染流程的影响：**

```
传统加载：
HTML 解析 → CSSOM → DOM + CSSOM → Render Tree → Layout → Paint → Composite
    ↓
所有资源加载（阻塞）

懒加载：
HTML 解析 → CSSOM → DOM + CSSOM → Render Tree → Layout → Paint → Composite
    ↓
首屏资源加载（非阻塞）
    ↓ (用户滚动后)
懒加载资源触发 → 异步加载 → 更新 DOM → Paint → Composite
```

## 4. 基础用法

### 4.1 图片懒加载

**方式一：原生 HTML（推荐）**

现代浏览器原生支持 `loading="lazy"` 属性：

```html
<!-- 关键图片：立即加载 -->
<img src="hero.jpg" alt="Hero" />

<!-- 非关键图片：懒加载 -->
<img src="long-list-item.jpg" alt="Item" loading="lazy" />
```

**优点：** 简单、无需第三方库、浏览器原生支持

**兼容性：** Chrome 76+、Firefox 75+、Edge 79+

---

**方式二：Intersection Observer API**

```javascript
// 创建观察器
const imageObserver = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      img.classList.remove('lazy');
      imageObserver.unobserve(img);
    }
  });
});

// 观察所有懒加载图片
document.querySelectorAll('img.lazy').forEach(img => {
  imageObserver.observe(img);
});
```

```html
<img class="lazy" data-src="image.jpg" alt="Lazy Image" />
```

**方式三：传统 scroll 事件 + 节流**

```javascript
// 节流函数
function throttle(func, delay) {
  let lastCall = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastCall >= delay) {
      lastCall = now;
      func.apply(this, args);
    }
  };
}

// 检查元素是否可见
function isVisible(element) {
  const rect = element.getBoundingClientRect();
  return (
    rect.top < window.innerHeight &&
    rect.bottom > 0
  );
}

// 懒加载函数
function lazyLoad() {
  const lazyImages = document.querySelectorAll('img.lazy');
  lazyImages.forEach(img => {
    if (isVisible(img)) {
      img.src = img.dataset.src;
      img.classList.remove('lazy');
    }
  });
}

// 监听滚动
window.addEventListener('scroll', throttle(lazyLoad, 100));
lazyLoad(); // 首次加载
```

## 5. 高级特性

### 5.1 预加载策略

**问题：** 懒加载可能导致用户滚动时看到空白。

**解决方案：** 使用 `rootMargin` 提前预加载。

```javascript
// 提前 200px 开始加载
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.src = entry.target.dataset.src;
      observer.unobserve(entry.target);
    }
  });
}, {
  rootMargin: '200px 0px' // 视口外 200px 就开始加载
});
```

### 5.4 无限滚动 (Infinite Scroll)

```jsx
import { useEffect, useRef, useState } from 'react';

function InfiniteScrollList({ fetchMore }) {
  const [items, setItems] = useState([]);
  const [page, setPage] = useState(1);
  const [hasMore, setHasMore] = useState(true);
  const loaderRef = useRef();

  // 使用 Intersection Observer 检测滚动到底部
  useEffect(() => {
    const observer = new IntersectionObserver(
      entries => {
        if (entries[0].isIntersecting && hasMore) {
          setPage(p => p + 1);
        }
      },
      { threshold: 0.1 }
    );

    if (loaderRef.current) {
      observer.observe(loaderRef.current);
    }

    return () => observer.disconnect();
  }, [hasMore]);

  // 加载更多数据
  useEffect(() => {
    fetchMore(page).then(newItems => {
      if (newItems.length === 0) {
        setHasMore(false);
      } else {
        setItems(prev => [...prev, ...newItems]);
      }
    });
  }, [page, fetchMore]);

  return (
    <div>
      {items.map(item => (
        <div key={item.id}>{item.content}</div>
      ))}
      <div ref={loaderRef}>
        {hasMore ? '加载中...' : '没有更多了'}
      </div>
    </div>
  );
}
```

## 6. 实战案例

### 6.1 电商商品列表优化

**场景：** 商品列表页有上百个商品图片，首屏加载缓慢。

**优化方案：**

```jsx
import { useEffect, useRef } from 'react';

function ProductList({ products }) {
  const imageRefs = useRef([]);

  useEffect(() => {
    const imageObserver = new IntersectionObserver(
      (entries) => {
        entries.forEach(entry => {
          if (entry.isIntersecting) {
            const img = entry.target;
            img.src = img.dataset.src;
            imageObserver.unobserve(img);
          }
        });
      },
      { rootMargin: '50px 0px' } // 提前 50px 加载
    );

    imageRefs.current.forEach(img => {
      if (img) imageObserver.observe(img);
    });

    return () => imageObserver.disconnect();
  }, []);

  return (
    <div className="product-grid">
      {products.map((product, index) => (
        <div key={product.id} className="product-card">
          <img
            ref={el => imageRefs.current[index] = el}
            data-src={product.imageUrl}
            src="placeholder.jpg"
            alt={product.name}
            width="200"
            height="200"
          />
          <h3>{product.name}</h3>
          <p>${product.price}</p>
        </div>
      ))}
    </div>
  );
}
```

### 6.3 社交媒体 Feed 流

**场景：** 无限滚动的社交媒体动态流。

**实现方案：**

```jsx
import { useInView } from 'react-intersection-observer';
import { useEffect, useState } from 'react';

function Feed() {
  const [posts, setPosts] = useState([]);
  const [page, setPage] = useState(1);

  const { ref: loadMoreRef, inView } = useInView({
    threshold: 0.1
  });

  // 加载更多帖子
  useEffect(() => {
    if (inView) {
      setPage(p => p + 1);
    }
  }, [inView]);

  // 获取帖子数据
  useEffect(() => {
    fetch(`/api/posts?page=${page}`)
      .then(res => res.json())
      .then(data => {
        setPosts(prev => [...prev, ...data]);
      });
  }, [page]);

  return (
    <div>
      {posts.map(post => (
        <PostCard key={post.id} post={post} />
      ))}

      {/* 加载触发器 */}
      <div ref={loadMoreRef} className="loading-trigger">
        加载中...
      </div>
    </div>
  );
}
```

## 7. 常见问题

### 7.1 布局抖动 (CLS) 问题

**问题表现：** 图片加载后页面突然跳动。

**原因：** 图片加载前后尺寸不一致。

**解决方案：**

```html
<!-- 方式一：固定宽高 -->
<img src="placeholder.jpg" width="300" height="200" />

<!-- 方式二：aspect-ratio -->
<div style="aspect-ratio: 16/9;">
  <img data-src="image.jpg" style="width: 100%; height: 100%;" />
</div>

<!-- 方式三：骨架屏占位 -->
<div class="skeleton" style="height: 200px;"></div>
```

### 7.2 SSR 水合不匹配

**问题表现：** Next.js 报错 "Text content does not match"。

**原因：** 服务端和客户端渲染内容不一致。

**解决方案：**

```jsx
// 使用 useEffect 延迟渲染
function ClientOnly({ children }) {
  const [mounted, setMounted] = useState(false);

  useEffect(() => setMounted(true), []);

  if (!mounted) return null;
  return children;
}

// 使用方式
<ClientOnly>
  <LazyComponent />
</ClientOnly>
```

### 7.3 图片加载失败处理

**问题：** 懒加载图片可能 404 或加载失败。

**解决方案：**

```jsx
function LazyImage({ src, placeholder, alt }) {
  const [imgSrc, setImgSrc] = useState(placeholder);
  const [error, setError] = useState(false);

  return (
    <img
      src={imgSrc}
      data-src={src}
      alt={alt}
      onError={() => {
        setError(true);
        setImgSrc('/images/error-placeholder.png');
      }}
      onLoad={() => setImgSrc(src)}
    />
  );
}
```

### 7.4 Intersection Observer 兼容性

**问题：** 旧浏览器不支持 Intersection Observer。

**解决方案：**

```javascript
// 检测支持性
if ('IntersectionObserver' in window) {
  // 使用 Intersection Observer
  const observer = new IntersectionObserver(...);
} else {
  // 降级到 scroll 监听
  window.addEventListener('scroll', throttle(lazyLoad, 100));
}

// 或使用 polyfill
import 'intersection-observer';
```
