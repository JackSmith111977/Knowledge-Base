## 二、理解过程

### 初始理解

### 发现的盲点

| 盲点 | 说明 |
|------|------|
| margin 三值简写 | 误以为三值 `margin: 10px 20px 30px` 中左侧 = 30px，实际是 `上 / 左右 / 下`，左侧 = 20px |
| Flex `margin: auto` 推底 | 混淆了 `align-self`（交叉轴对齐）和 `margin: auto`（吸收剩余空间）的作用机制 |
| `minmax()` 最小值约束 | 误以为 min 值不满足时布局会失败不渲染，实际是遵守 min 约束导致溢出 |
| `animation-fill-mode` | 混淆了 `forwards`（保持最终帧）和 `infinite`（无限循环） |

### 文档纠错（学习过程中发现并修正）

> **flex-shrink 计算错误**：文档原文采用简化均分算法（`每份收缩 = 溢出量 / 项目数`），
> 但 CSS 规范明确规定收缩量应按 **shrink × basis 加权比例**分配。
> 已修正文档中的两处计算示例，更新为规范公式：
> `每项收缩量 = 溢出量 × (shrink × basis) / Σ(shrink × basis)`

### 知识辨析（主动提问后澄清）

> **脱离文档流 ≠ 创建层叠上下文**：两个正交概念。
> - `position: absolute; z-index: auto` 脱离文档流但不创建层叠上下文
> - `opacity: 0.9` 不脱离文档流但创建层叠上下文

## 四、错题本

| # | 章节 | 知识点 | 错误详情 | 正确理解 |
|---|------|--------|----------|----------|
| 1 | 4. 盒模型 | margin 多值简写规则 | `margin: 10px 20px 30px` 误以为左侧 = 30px | 三值写法：`上 / 左右 / 下`，左侧 = 中间值 20px |
| 2 | 6. Flex 布局 | `margin: auto` 推底技巧 | 误选 `align-self: flex-end` 为推底方式 | `margin-top: auto` 吸收上方剩余空间，推元素到底部；`align-self` 是交叉轴对齐 |
| 3 | 7. Grid 布局 | `minmax()` 最小值约束 | 误以为 min 不满足时布局失败 | 浏览器遵守 min 约束，总宽度超出容器时会溢出而非失败 |
| 4 | 9. 动画 | `animation-fill-mode` | 误选 `infinite` 为保持最终状态 | `forwards` 保持最终帧，`infinite` 是无限循环永不停止 |

## 五、核心要点

1. **CSS 层叠是完整决策流程**：重要性 → 来源 → 特异性 → 书写顺序，优先级只是其中一环
2. **盒模型全局统一 `border-box`** 是现代开发的最佳实践
3. **Flex 是一维布局，Grid 是二维布局**，两者互补而非替代
4. **flex-shrink 按 shrink × basis 加权分配**，不是简单均分
5. **动画仅用 `transform` 和 `opacity`** 才能保证 60fps 流畅体验
6. **`will-change` 是提示而非保证**，用完需释放，避免内存浪费
7. **移动优先 = `min-width` 从小往大**，代码更简洁
8. **`clamp()` 实现流体排版**，但复杂布局变化仍需媒体查询
