# 组 A 任务说明（基础章）

## 第 1 章 - 软件架构概述
**调研目标：**
1. 软件架构的完整定义和范畴
2. 软件架构的重要性（为什么需要架构）
3. 架构活动的核心内容
4. 架构历史演变

**内容要求：**
- 概念定义：准确引用权威来源
- 工作原理：架构如何指导系统开发
- 示例：架构决策的实际案例
- 误区：常见误解

## 第 2 章 - 核心概念详解
**调研目标：**
1. 软件元素、关系、属性的定义
2. 利益相关者及其关切
3. 关注点分离原理
4. 架构决策与理由

**内容要求：**
- 每个概念独立小节
- 概念之间的关系说明
- Mermaid 图展示概念关联
