## 1. 基础认知与链表结构

### 1.1 什么是链表

**定义**：链表是由一系列节点组成的线性数据结构，每个节点包含**数据域**和**指针域**。与数组不同，链表的节点在内存中不连续存储，通过指针链接形成逻辑上的线性序列。

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
```

### 1.2 链表 vs 数组

| 维度 | 数组 | 链表 |
|------|------|------|
| 内存布局 | 连续 | 不连续 |
| 随机访问 | O(1) | O(n) |
| 头部插入/删除 | O(n)（需移动） | O(1) |
| 中间插入/删除 | O(n)（需移动） | O(1)（已知位置） |
| 空间开销 | 紧凑 | 每个节点额外一个指针 |
| 缓存友好性 | 好（局部性原理） | 差（节点分散） |

### 1.3 链表类型

| 类型 | 结构 | 特点 |
|------|------|------|
| 单向链表 | `A → B → C → null` | 只能单向遍历 |
| 双向链表 | `null ← A ↔ B ↔ C → null` | 可前后双向遍历 |
| 循环链表 | `A → B → C → A` | 尾节点指向头节点 |

### 1.4 链表操作的时间复杂度

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 遍历 | O(n) | O(1) |
| 查找 | O(n) | O(1) |
| 头部插入 | O(1) | O(1) |
| 尾部插入 | O(n) / O(1)（有尾指针） | O(1) |
| 中间插入 | O(1)（已知位置） | O(1) |
| 头部删除 | O(1) | O(1) |
| 中间删除 | O(1)（已知位置） | O(1) |

### 1.5 链表的"盒子-标签-纸条"模型

理解链表操作的核心比喻：

- **盒子**：节点在内存中的位置（不变）
- **标签**：指针变量（prev、curr、slow、fast 等），指向不同节点
- **纸条**：节点的 `next` 指针，决定节点之间的连接关系

> 链表翻转的本质：剪断绳子（`next`），拴到另一个盒子上。盒子不动，标签挪，纸条改了链就变。

## 2. 遍历与原地修改技巧

### 2.1 遍历模板

```python
# 基础遍历
curr = head
while curr:
    # 处理 curr
    curr = curr.next
```

### 2.2 原地修改的核心原则

**不要丢失后继引用**：在修改 `curr.next` 之前，必须先保存原来的后继节点。

```python
# 错误写法 — 丢失了 curr.next
curr.next = prev
curr = curr.next  # ❌ curr.next 已经被改了！

# 正确写法 — 暂存后继
next_temp = curr.next
curr.next = prev
curr = next_temp  # ✅ 正确移动到下一个节点
```

### 2.3 原地修改常见场景

**删除当前节点**（已知不是尾节点）：
```python
# 用后继的值覆盖自己，然后删除后继
curr.val = curr.next.val
curr.next = curr.next.next
```

**删除下一个节点**：
```python
curr.next = curr.next.next  # 跳过下一个节点
```

### 2.4 常见误区

| 误区 | 错误 | 正确 |
|------|------|------|
| 空链表未处理 | 直接 `head.next` | 开头判断 `if not head` |
| 单节点未处理 | `head.next.next` | 判断 `if not head.next` |
| 修改指针后丢失引用 | `curr.next = prev; curr = curr.next` | 先保存 `next_temp = curr.next` |

## 3. 虚拟头节点（Dummy Node）

### 3.1 什么是 Dummy Node

**定义**：在链表真实头节点之前添加一个无意义的哨兵节点，其 `next` 指向真实头节点。

```python
dummy = ListNode(0)
dummy.next = head
```

### 3.2 为什么需要 Dummy Node

**核心问题**：链表的头节点在操作中可能被修改或删除，导致需要特殊处理边界情况。

**不用 Dummy 的删除元素代码**（需要特殊处理头节点）：
```python
def removeElements(head, target):
    while head and head.val == target:  # 特殊处理头节点
        head = head.next
    curr = head
    while curr and curr.next:
        if curr.next.val == target:
            curr.next = curr.next.next
        else:
            curr = curr.next
    return head
```

**使用 Dummy 后的代码**（头节点和中间节点统一处理）：
```python
def removeElements(head, target):
    dummy = ListNode(0)
    dummy.next = head
    curr = dummy
    while curr.next:
        if curr.next.val == target:
            curr.next = curr.next.next
        else:
            curr = curr.next
    return dummy.next  # dummy.next 始终是新头节点
```

### 3.3 Dummy Node 的使用场景

| 场景 | 典型题 | 为什么需要 |
|------|--------|-----------|
| 头节点可能被删除 | 203 移除链表元素 | 删除后不知道新头是谁 |
| 头节点可能被移动 | 82 删除重复元素II | 重复元素可能包含头节点 |
| 需要返回新头节点 | 21 合并两个有序链表 | 新头不确定是哪个 |
| 需要维护前驱节点 | 19 删除倒数第N个 | 删除节点的前驱可能是头 |

### 3.4 Dummy Node 的核心价值

> **让"删除头节点"和"删除中间节点"、"创建新链表头"和"拼接后续节点"的逻辑完全统一。** 最终返回 `dummy.next` 即可。

### 3.5 常见误区

| 误区 | 说明 |
|------|------|
| 忘记 `return dummy.next` | 返回了 dummy 本身而不是新头 |
| dummy 的值有意义 | dummy 的 val 应该无关紧要，只用它的 next |
| 忘记释放 dummy（C++） | 非 GC 语言需要 `delete dummy` |

## 4. 反转链表（核心专题）

### 4.1 迭代法 — 三指针法

**原理**：逐个反转每个节点的 `next` 指针，使其指向原来的前驱节点。

```python
def reverseList(head):
    prev = None        # 已反转区间的尾部
    curr = head        # 当前处理节点
    while curr:
        next_temp = curr.next  # ① 暂存后继（防止断链）
        curr.next = prev       # ② 反转指向
        prev = curr            # ③ prev 前进
        curr = next_temp       # ④ curr 前进
    return prev                # prev 是新头节点
```

**源码级解析**：

以 `1 → 2 → 3 → null` 为例，追踪每一轮迭代后的状态：

```
初始：prev=None, curr=1→2→3→null

第1轮：
  next_temp = 2→3→null     # 暂存后继
  1.next = None            # 反转：1 指向 null
  prev = 1→null            # prev 前进
  curr = 2→3→null          # curr 前进

第2轮：
  next_temp = 3→null
  2.next = 1→null          # 反转：2 指向 1
  prev = 2→1→null
  curr = 3→null

第3轮：
  next_temp = null
  3.next = 2→1→null        # 反转：3 指向 2
  prev = 3→2→1→null        # ✅ 新头节点
  curr = null              # 循环结束

返回 prev = 3→2→1→null
```

**时间复杂度**：O(n)，每个节点访问一次
**空间复杂度**：O(1)，只用了三个指针

### 4.2 递归法

**原理**：先递归反转后面的节点，再让当前节点的后继指向自己。

```python
def reverseList(head):
    # base case：空链表或单节点，不需要反转
    if not head or not head.next:
        return head
    
    newHead = reverseList(head.next)  # ① 先反转后面所有
    head.next.next = head             # ② 让后继指向自己
    head.next = None                  # ③ 断开自己的 next（防止成环）
    
    return newHead                    # 新头节点始终是原链表的尾节点
```

**源码级解析**：

以 `1 → 2 → 3 → null` 为例，追踪递归调用栈：

```
reverseList(1→2→3→null)
  → reverseList(2→3→null)
    → reverseList(3→null)
      → base case: 返回 3→null（newHead = 3）
    ← head = 2: head.next.next = head → 3.next = 2
                head.next = None     → 2.next = null
      返回 3→2→null
  ← head = 1: head.next.next = head → 2.next = 1
              head.next = None     → 1.next = null
    返回 3→2→1→null
```

**递归 vs 迭代对比**：

| 维度 | 迭代 | 递归 |
|------|------|------|
| 时间 | O(n) | O(n) |
| 空间 | O(1) | O(n)（调用栈） |
| 可读性 | 中等 | 更简洁 |
| 面试推荐 | ✅ 首选 | 加分项 |
| 栈溢出风险 | 无 | n > 递归深度限制时溢出 |

### 4.3 反转链表 II（区间反转）

**题目**：[92] 反转从位置 left 到 right 的链表

**核心思路**：先用 Dummy + 遍历定位到 left 前一个位置，然后对区间内执行标准反转。

```python
def reverseBetween(head, left, right):
    dummy = ListNode(0)
    dummy.next = head
    p0 = dummy
    
    # 定位到 left 的前一个位置
    for _ in range(left - 1):
        p0 = p0.next
    
    # 对 [left, right] 区间执行标准反转
    prev = None
    curr = p0.next
    for _ in range(right - left + 1):
        next_temp = curr.next
        curr.next = prev
        prev = curr
        curr = next_temp
    
    # 重新连接
    p0.next.next = curr   # 原区间头部（现为尾部）指向区间后第一个节点
    p0.next = prev        # p0 指向新区间头部（原区间尾部）
    
    return dummy.next
```

### 4.4 K 个一组翻转链表

**题目**：[25] 每 k 个节点一组翻转

**核心思路**：迭代遍历每一组，每组执行标准反转，不足 k 个的不反转。

```python
def reverseKGroup(head, k):
    dummy = ListNode(0)
    dummy.next = head
    p0 = dummy  # p0 始终站在当前组的前一个位置
    
    curr = head
    while True:
        # 先走 k-1 步，检查是否够 k 个节点
        tail = p0
        for _ in range(k):
            tail = tail.next
            if not tail:
                return dummy.next  # 不足 k 个，直接返回
        
        # 保存下一组的起点
        next_group = tail.next
        
        # 对当前组执行标准反转（head 到 tail）
        prev, curr_rev = None, p0.next
        for _ in range(k):
            next_temp = curr_rev.next
            curr_rev.next = prev
            prev = curr_rev
            curr_rev = next_temp
        
        # 重新连接
        p0.next.next = next_group  # 原组头部指向下一组起点
        p0.next = tail             # p0 指向新组头（原组尾部）
        p0 = p0.next               # p0 前进到当前组尾部（下一组前）
    
    return dummy.next
```

**口诀**："先数数，够数就翻，翻完缝合。"

### 4.5 两两交换链表中的节点

**题目**：[24] 每两个相邻节点交换

这是 K=2 的特殊情况，可以直接用模板：

```python
def swapPairs(head):
    dummy = ListNode(0)
    dummy.next = head
    p0 = dummy
    
    while p0.next and p0.next.next:
        # 当前组的两个节点
        a = p0.next
        b = p0.next.next
        
        # 交换 a 和 b
        p0.next = b
        a.next = b.next
        b.next = a
        
        # p0 前进到 a（交换后 a 是组的尾部）
        p0 = a
    
    return dummy.next
```

## 5. 双指针技术

### 5.1 前后指针（相向双指针）

**原理**：两个指针从不同起点出发，保持固定距离或不同速度移动。

**典型应用 — 删除倒数第 N 个节点**：

```python
def removeNthFromEnd(head, n):
    dummy = ListNode(0)
    dummy.next = head
    fast = dummy
    slow = dummy
    
    # fast 先走 n+1 步（+1 因为 dummy）
    for _ in range(n + 1):
        fast = fast.next
    
    # fast 和 slow 同时前进
    while fast:
        fast = fast.next
        slow = slow.next
    
    # slow 此时在倒数第 N+1 个位置
    slow.next = slow.next.next
    
    return dummy.next
```

**为什么 fast 先走 n+1 步**：因为用了 dummy，slow 最终停在被删除节点的前一个位置。

### 5.2 同向双指针

**原理**：两个指针同向移动，一个标记读取位置，一个标记写入位置。

**典型应用 — 分隔链表（[86]）**：

```python
def partition(head, x):
    # 创建两个虚拟头节点，分别存小于 x 和大于等于 x 的节点
    small = ListNode(0)
    large = ListNode(0)
    smallTail = small
    largeTail = large
    
    curr = head
    while curr:
        if curr.val < x:
            smallTail.next = curr
            smallTail = smallTail.next
        else:
            largeTail.next = curr
            largeTail = largeTail.next
        curr = curr.next
    
    # 拼接
    largeTail.next = None        # 防止成环
    smallTail.next = large.next
    return small.next
```

### 5.3 背向双指针

**原理**：从某个中心点向两边扩展。

**典型应用**：[976] 三角形的最大周长（排序后从右向左检查连续三个数）

## 6. 快慢指针原理

### 6.1 Floyd 判圈算法（龟兔赛跑）

**核心思想**：慢指针每次走 1 步，快指针每次走 2 步。如果有环，快指针一定会追上慢指针。

```python
def hasCycle(head):
    slow = head
    fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True  # 有环
    return False  # fast 到达末尾，无环
```

### 6.2 为什么慢指针一定能追上快指针？

**数学证明**：

设：
- 起点到环入口的距离为 `a`
- 环入口到相遇点的距离为 `b`
- 环的长度为 `c`
- 快指针在环内绕了 `k` 圈

相遇时：
- 慢指针走了 `a + b` 步
- 快指针走了 `a + b + k * c` 步
- 快指针速度是慢指针的 2 倍

```
2 * (a + b) = a + b + k * c
a + b = k * c
a = k * c - b
```

这个等式说明：**从起点走 `a` 步到环入口，等于从相遇点走 `a` 步（绕环 `k-1` 圈后再走 `c-b` 步）也到环入口**。

**更直观的理解**：快指针相对于慢指针的速度是 `2 - 1 = 1` 步/轮。在环内，快慢指针之间的距离最多为环的长度 `c`。因为相对速度是 1，而 1 能整除任何整数，所以快指针一定会在 `c` 轮内追上慢指针。

> **关键结论**：只要相对速度为 1，环的长度为任意正整数 c，快指针最多绕环一圈就能追上慢指针。

### 6.3 找环入口（[142] 环形链表 II）

```python
def detectCycle(head):
    slow = head
    fast = head
    
    # 第一阶段：判断有环
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            break
    else:
        return None  # 无环
    
    # 第二阶段：找环入口
    slow = head  # 慢指针重置到起点
    while slow != fast:
        slow = slow.next
        fast = fast.next  # 注意：这次每次只走 1 步
    
    return slow  # 再次相遇的点就是环入口
```

### 6.4 找链表中点

```python
def middleNode(head):
    slow = head
    fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow
```

**奇偶长度差异**：
- 奇数长度：slow 停在正中间
- 偶数长度：slow 停在右中点（后半部分的第一个节点）

### 6.5 回文链表（[234]）

```python
def isPalindrome(head):
    # 步骤 1：找中点
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    
    # 步骤 2：反转后半部分
    prev = None
    while slow:
        next_temp = slow.next
        slow.next = prev
        prev = slow
        slow = next_temp
    
    # 步骤 3：比较前后两半
    left, right = head, prev
    while right:  # 后半部分可能更短（奇数长度时）
        if left.val != right.val:
            return False
        left = left.next
        right = right.next
    
    return True
```

## 7. 合并、分治与设计题

### 7.1 合并两个有序链表（[21]）

```python
def mergeTwoLists(l1, l2):
    dummy = ListNode(0)
    tail = dummy
    
    while l1 and l2:
        if l1.val < l2.val:
            tail.next = l1
            l1 = l1.next
        else:
            tail.next = l2
            l2 = l2.next
        tail = tail.next
    
    # 拼接剩余部分（只有一个非空）
    tail.next = l1 if l1 else l2
    
    return dummy.next
```

### 7.2 合并 K 个有序链表（[23]）

**分治法**（推荐）：
```python
def mergeKLists(lists):
    if not lists:
        return None
    
    def merge(left, right):
        if left == right:
            return lists[left]
        mid = (left + right) // 2
        l1 = merge(left, mid)
        l2 = merge(mid + 1, right)
        return mergeTwoLists(l1, l2)
    
    return merge(0, len(lists) - 1)
```

**时间复杂度**：O(n log k)，其中 n 是总节点数，k 是链表数
**空间复杂度**：O(log k)（递归栈）

### 7.3 排序链表（[148]）

归并排序是唯一能在 O(n log n) 时间内对链表排序的算法：

```python
def sortList(head):
    if not head or not head.next:
        return head
    
    # 找中点（用快慢指针）
    slow = fast = head
    while fast.next and fast.next.next:
        slow = slow.next
        fast = fast.next.next
    
    # 断开
    mid = slow.next
    slow.next = None
    
    # 递归排序 + 合并
    left = sortList(head)
    right = sortList(mid)
    return mergeTwoLists(left, right)
```

### 7.4 LRU 缓存（[146]）

链表 + 哈希表的经典设计题：

```python
class LRUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.cache = {}
        self.head = ListNode(0)  # 虚拟头
        self.tail = ListNode(0)  # 虚拟尾
        self.head.next = self.tail
        self.tail.prev = self.head
    
    def _remove(self, node):
        prev, nxt = node.prev, node.next
        prev.next = nxt
        nxt.prev = prev
    
    def _add_to_head(self, node):
        node.next = self.head.next
        node.prev = self.head
        self.head.next.prev = node
        self.head.next = node
```

## 8. 常见陷阱与刷题路线

### 8.1 链表操作常见陷阱

| 陷阱 | 说明 | 预防 |
|------|------|------|
| **断链** | 修改 `next` 后丢失后继引用 | 修改前先保存 `next_temp = curr.next` |
| **空指针** | 对 null 调用 `.next` | 循环条件检查 `while curr and curr.next` |
| **成环** | 修改指针后形成死循环 | 尾部操作后设置 `tail.next = None` |
| **头节点变化** | 删除/反转后不知道新头 | 使用 Dummy Node |
| **奇偶长度** | 找中点时奇偶长度行为不同 | 明确 fast 终止条件 |
| **单节点** | 忘记处理只有一个节点的情况 | base case 检查 `if not head.next` |

### 8.2 链表调试技巧

1. **打印链表**：写一个辅助函数 `printList(head)`
2. **画图追踪**：每个指针操作用纸笔画出 prev/curr/next 的指向
3. **Dummy 调试**：加上 Dummy 后，用 `dummy.next` 始终能看到当前链表全貌
