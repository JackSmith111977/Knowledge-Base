## 🔑 核心知识要点

### 1. Locator 策略优先级
```
getByTestId (推荐) > getByRole (语义化) > getByText > CSS > XPath
```

### 2. 自动等待检查项
- Attached（存在于 DOM）
- Visible（可见）
- Stable（稳定，无动画）
- Enabled（已启用）
- Editable（可编辑，fill 专用）
- Interceptable（可交互，无遮挡）

### 3. 断言自动重试
```typescript
// ✅ 自动重试
await expect(page.locator('#count')).toHaveText('5');

// ❌ 不重试
expect(await page.textContent('#count')).toBe('5');
```

### 4. Trace Viewer 配置
```typescript
use: {
  trace: 'on-first-retry'  // 仅在第一次重试时录制
}
```

### 5. Page Object 模式
- 封装页面元素和操作
- 提高测试可维护性
- 减少代码重复

## 💡 关键洞见

1. **自动等待是 Playwright 的核心竞争力**：解决了传统测试框架的时序问题痛点

2. **Trace Viewer 是调试神器**：比截图/录屏提供更丰富的上下文信息

3. **测试隔离是关键**：使用独立上下文确保测试可并行、可重现

4. **Locator 选择影响可维护性**：getByTestId 最稳定，但需要开发配合添加
