#### 1. DOCTYPE 的核心作用

**问题**：`<!DOCTYPE html>` 的作用是什么？

**完整答案**：

DOCTYPE 的核心作用是**触发浏览器的标准模式渲染**，防止进入**怪异模式（Quirks Mode）**。

**标准模式 vs 怪异模式：**

| 特性 | 标准模式 | 怪异模式 |
|------|----------|----------|
| 触发条件 | 有正确的 DOCTYPE | 缺少或错误的 DOCTYPE |
| 盒模型 | W3C 标准盒模型 | IE 盒模型（width 包含 padding） |
| 行内元素垂直对齐 | baseline | 底部对齐 |
| 表格单元格垂直对齐 | baseline | 基线对齐 |
| 图片底部间距 | 有间距 | 无间距 |

```html
<!-- 正确：HTML5 DOCTYPE -->
<!DOCTYPE html>

<!-- 错误：会导致怪异模式 -->
<!-- 缺少 DOCTYPE -->
<!-- 或 <!DOCTYPE html SYSTEM "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> -->
```

**完整 HTML5 模板：**
```html
<!DOCTYPE html>  <!-- 必须是第一行，触发标准模式 -->
<html lang="zh-CN">  <!-- lang 属性有助于搜索引擎和屏幕阅读器 -->
<head>
    <meta charset="UTF-8">  <!-- 字符编码，防止中文乱码 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">  <!-- 移动端适配 -->
    <meta name="description" content="页面描述（150-160 字符）">  <!-- SEO -->
    <title>页面标题</title>
    <link rel="stylesheet" href="styles.css">
    <link rel="icon" href="/favicon.ico">
</head>
<body>
    <!-- 页面内容 -->
</body>
</html>
```

#### 2. HTML5 新增 input 类型

**问题**：HTML5 为 `<input>` 标签新增了哪些类型？

**完整答案**：

**HTML5 新增的 input 类型（共 13 种）：**

```html
<!-- 联系方式类（自动验证格式） -->
<input type="email" placeholder="邮箱">
<!-- 自动验证格式：xxx@xxx.xxx -->

<input type="tel" placeholder="手机号">
<!-- 移动端会显示数字键盘 -->

<input type="url" placeholder="网址">
<!-- 自动验证格式：http(s)://xxx -->

<!-- 日期时间类（显示原生选择器） -->
<input type="date">
<!-- 显示日期选择器，返回 YYYY-MM-DD -->

<input type="time">
<!-- 显示时间选择器，返回 HH:MM -->

<input type="datetime-local">
<!-- 本地日期时间，返回 YYYY-MM-DDTHH:MM -->

<input type="month">
<!-- 月份选择，返回 YYYY-MM -->

<input type="week">
<!-- 周选择，返回 YYYY-Www -->

<!-- 数值与范围 -->
<input type="number" min="0" max="100" step="1">
<!-- 带上下箭头的数字输入框 -->

<input type="range" min="0" max="100" step="1">
<!-- 滑动条 -->

<!-- 其他 -->
<input type="search">
<!-- 搜索框，带清除按钮，iOS 上圆角 -->

<input type="color">
<!-- 颜色选择器，返回 #RRGGBB -->
```

**移动端键盘优化：**
```html
<!-- 不同触发发的软键盘类型 -->
<input type="email">     <!-- @ 符号键盘 -->
<input type="tel">       <!-- 数字键盘 -->
<input type="number">    <!-- 数字键盘（带 +/-） -->
<input type="url">       <!-- . / : 键盘 -->
```

**不是 HTML5 新增的类型（常考）：**
```html
<input type="text">      <!-- HTML4 就有 -->
<input type="password">  <!-- HTML4 就有 -->
<input type="checkbox">  <!-- HTML4 就有 -->
<input type="radio">     <!-- HTML4 就有 -->
<input type="file">      <!-- HTML4 就有 -->
<input type="hidden">    <!-- HTML4 就有 -->
<input type="image">     <!-- HTML4 就有 -->
<input type="button">    <!-- HTML4 就有 -->
<input type="submit">    <!-- HTML4 就有 -->
<input type="reset">     <!-- HTML4 就有 -->
```

#### 3. pattern 属性

**问题**：`pattern` 属性是什么？如何使用？

**完整答案**：

**`pattern` 属性用于正则表达式验证**，仅在 `type="text"、"password"、"email"、"url"、"tel"、"search"` 时生效。

```html
<!-- 基本语法 -->
<input type="text" pattern="正则表达式" title="错误提示">

<!-- 必填 + 模式匹配 -->
<input type="text" required pattern="[A-Za-z]{3,}" title="请输入至少 3 个字母">
```

**常用正则表达式示例：**

```html
<!-- 手机号（11 位数字） -->
<input type="tel" pattern="[1][3-9][0-9]{9}" title="请输入正确的手机号">

<!-- 身份证号 -->
<input type="text" pattern="[1-9][0-9]{5}(18|19|20)[0-9]{2}(0[1-9]|1[0-2])(0[1-9]|[12][0-9]|3[01])[0-9]{3}[0-9Xx]" title="请输入正确的身份证号">

<!-- 邮箱 -->
<input type="email" pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$" title="请输入正确的邮箱地址">

<!-- URL -->
<input type="url" pattern="https?://.+" title="请输入有效的 URL（http 或 https 开头）">

<!-- 密码（至少 8 位，包含字母和数字） -->
<input type="password" pattern="(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,}" title="密码至少 8 位，包含字母和数字">

<!-- 数字范围（1-100） -->
<input type="text" pattern="([1-9]|[1-9][0-9]|100)" title="请输入 1 到 100 的数字">

<!-- 中文姓名（2-10 个汉字） -->
<input type="text" pattern="[\u4e00-\u9fa5]{2,10}" title="请输入 2-10 个汉字的姓名">
```

**与 JavaScript 配合使用：**
```javascript
const input = document.querySelector('input');

// 监听输入，实时验证
input.addEventListener('input', () => {
    if (input.validity.patternMismatch) {
        input.setCustomValidity('格式不匹配');
    } else {
        input.setCustomValidity('');
    }
});

// 表单提交时
input.form.addEventListener('submit', (e) => {
    if (!input.checkValidity()) {
        e.preventDefault();
        input.reportValidity();  // 显示错误提示
    }
});
```

#### 4. `<video>` 标签详解

**问题**：`<video>` 标签有哪些常用属性？如何实现跨浏览器兼容？

**完整答案**：

**`<video>` 常用属性：**

```html
<video
    controls           <!-- 显示播放控制条（播放/暂停/音量/全屏） -->
    autoplay           <!-- 自动播放（注意：Chrome/Safari 要求静音） -->
    loop               <!-- 循环播放 -->
    muted              <!-- 静音 -->
    preload="metadata" <!-- 预加载策略 -->
    width="640"        <!-- 宽度 -->
    height="360"       <!-- 高度 -->
    poster="cover.jpg" <!-- 封面图（加载前显示） -->
    playsinline        <!-- iOS 内联播放（不全屏） -->
>
    <!-- 多格式源 -->
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">

    <!-- 字幕轨道 -->
    <track src="subtitles.vtt" kind="subtitles" srclang="zh" label="中文">

    <!-- 降级提示 -->
    您的浏览器不支持视频播放。
</video>
```

**preload 属性值：**
| 值 | 说明 | 使用场景 |
|---|------|---------|
| `auto` | 加载整个视频 | 确定用户会播放 |
| `metadata` | 仅加载元数据（时长、尺寸） | 推荐，节省带宽 |
| `none` | 不预加载 | 用户可能不播放 |

**跨浏览器兼容方案：**

```html
<video controls width="640" poster="cover.jpg">
    <!-- MP4 (H.264) - Safari、IE、Edge 首选 -->
    <source src="video.mp4" type="video/mp4">

    <!-- WebM (VP9) - Chrome、Firefox 首选 -->
    <source src="video.webm" type="video/webm">

    <!-- OGG (Theora) - Firefox 备用 -->
    <source src="video.ogg" type="video/ogg">

    <!-- Flash 回退（极端情况） -->
    <object type="application/x-shockwave-flash" data="player.swf">
        <param name="movie" value="player.swf">
        <param name="flashvars" value="file=video.mp4">
        您的浏览器不支持视频播放。
    </object>
</video>
```

**浏览器格式支持表：**
| 格式 | 编码 | Chrome | Firefox | Safari | Edge |
|------|------|--------|---------|--------|------|
| MP4 | H.264 + AAC | ✓ | ✓ | ✓ | ✓ |
| WebM | VP9/VP8 + Vorbis | ✓ | ✓ | ✗ (部分) | ✓ |
| OGG | Theora + Vorbis | ✓ | ✓ | ✗ | ✗ |

**JavaScript API：**
```javascript
const video = document.querySelector('video');

// 播放控制
video.play();
video.pause();
video.currentTime = 30;  // 跳转到 30 秒

// 属性读取
video.duration;      // 总时长
video.currentTime;   // 当前时间
video.paused;        // 是否暂停
video.volume;        // 音量 (0-1)
video.muted;         // 是否静音
video.playbackRate;  // 播放倍速 (0.5, 1, 2)

// 事件监听
video.addEventListener('play', () => console.log('开始播放'));
video.addEventListener('pause', () => console.log('暂停'));
video.addEventListener('ended', () => console.log('播放结束'));
video.addEventListener('timeupdate', () => console.log('时间更新'));
video.addEventListener('waiting', () => console.log('缓冲中'));
video.addEventListener('canplay', () => console.log('可以播放'));
video.addEventListener('error', (e) => console.log('加载失败', e));
```

#### 5. 表单验证 API

**问题**：如何使用 JavaScript 触发表单的原生验证？

**完整答案**：

**表单验证 API 方法：**

```javascript
const form = document.querySelector('form');
const input = form.querySelector('input');

// 1. checkValidity() - 检查有效性，返回 true/false
// 不显示错误提示
if (input.checkValidity()) {
    console.log('输入有效');
} else {
    console.log('输入无效');
}

// 2. reportValidity() - 检查并显示错误提示
// 验证失败时浏览器会显示错误提示
if (!form.reportValidity()) {
    // 验证失败，可以执行其他逻辑
    return;
}

// 3. setCustomValidity() - 设置自定义错误信息
input.addEventListener('input', () => {
    if (input.value.length < 6) {
        input.setCustomValidity('密码至少需要 6 个字符');
    } else {
        input.setCustomValidity('');  // 清空错误信息
    }
});
```

**validity 对象属性：**
```javascript
input.validity.valid;           // 是否有效
input.validity.valueMissing;    // 必填项是否为空
input.validity.typeMismatch;    // 类型是否不匹配（如邮箱格式）
input.validity.patternMismatch; // 是否符合 pattern
input.validity.tooShort;        // 是否太短
input.validity.tooLong;         // 是否太长
input.validity.rangeUnderflow;  // 是否小于最小值
input.validity.rangeOverflow;   // 是否大于最大值
input.validity.stepMismatch;    // 是否符合 step
input.validity.badInput;        // 输入是否无效
```

**validationMessage 属性：**
```javascript
// 获取当前错误提示文本
console.log(input.validationMessage);

// 示例：自定义错误显示
const errorDiv = document.querySelector('.error');
if (!input.checkValidity()) {
    errorDiv.textContent = input.validationMessage;
    errorDiv.style.display = 'block';
}
```

**完整表单验证示例：**
```html
<form id="myForm" novalidate>
    <div>
        <label for="username">用户名</label>
        <input type="text" id="username" required minlength="3" maxlength="20">
        <span class="error"></span>
    </div>

    <div>
        <label for="password">密码</label>
        <input type="password" id="password" required minlength="6">
        <span class="error"></span>
    </div>

    <button type="submit">提交</button>
</form>

<script>
const form = document.getElementById('myForm');

form.addEventListener('submit', (e) => {
    e.preventDefault();

    const username = form.querySelector('#username');
    const password = form.querySelector('#password');
    let isValid = true;

    // 用户名验证
    if (!username.checkValidity()) {
        username.nextElementSibling.textContent = username.validationMessage;
        username.nextElementSibling.style.display = 'block';
        isValid = false;
    } else {
        username.nextElementSibling.style.display = 'none';
    }

    // 密码验证
    if (!password.checkValidity()) {
        password.nextElementSibling.textContent = password.validationMessage;
        password.nextElementSibling.style.display = 'block';
        isValid = false;
    } else {
        password.nextElementSibling.style.display = 'none';
    }

    if (isValid) {
        // 可以提交表单
        form.submit();
    }
});
</script>
```

#### 6. `<meta>` 标签与 Open Graph

**问题**：`<meta>` 标签有哪些常见用法？Open Graph 是什么？

**完整答案**：

**`<meta>` 标签常见用法：**

```html
<head>
    <!-- 字符编码（必须是 head 中第一个） -->
    <meta charset="UTF-8">

    <!-- 视口设置（移动端适配） -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- width=device-width: 宽度等于设备宽度 -->
    <!-- initial-scale=1.0: 初始缩放比例 -->
    <!-- maximum-scale=1.0: 最大缩放比例 -->
    <!-- user-scalable=no: 禁止用户缩放 -->

    <!-- SEO 描述（150-160 字符，显示在搜索结果中） -->
    <meta name="description" content="页面描述，吸引用户点击">

    <!-- 关键词（现代搜索引擎权重较低） -->
    <meta name="keywords" content="关键词 1，关键词 2，关键词 3">

    <!-- 作者 -->
    <meta name="author" content="作者名">

    <!-- 搜索引擎索引控制 -->
    <meta name="robots" content="index, follow">
    <!-- index: 允许索引 -->
    <!-- noindex: 禁止索引 -->
    <!-- follow: 跟随链接 -->
    <!-- nofollow: 不跟随链接 -->

    <!-- HTTP 响应头 -->
    <meta http-equiv="refresh" content="30">  <!-- 30 秒后刷新 -->
    <meta http-equiv="X-UA-Compatible" content="IE=edge">  <!-- IE 用最新引擎 -->
    <meta http-equiv="Content-Security-Policy" content="default-src 'self'">
</head>
```

**Open Graph（社交媒体优化）：**

Open Graph 是由 Facebook 提出的协议，让网页在社交媒体上分享时显示更丰富的预览。

```html
<head>
    <!-- Open Graph 基础属性 -->
    <meta property="og:type" content="website">
    <meta property="og:url" content="https://example.com/page">
    <meta property="og:title" content="页面标题">
    <meta property="og:description" content="页面描述">
    <meta property="og:image" content="https://example.com/thumbnail.jpg">
    <meta property="og:image:width" content="1200">
    <meta property="og:image:height" content="630">
    <meta property="og:site_name" content="网站名称">
    <meta property="og:locale" content="zh_CN">

    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary_large_image">
    <!-- summary: 小图 -->
    <!-- summary_large_image: 大图 -->

    <meta name="twitter:site" content="@twitter_handle">
    <meta name="twitter:title" content="页面标题">
    <meta name="twitter:description" content="页面描述">
    <meta name="twitter:image" content="https://example.com/thumbnail.jpg">
    <meta name="twitter:creator" content="@author_handle">

    <!-- 其他平台 -->
    <meta property="fb:app_id" content="123456789">  <!-- Facebook App ID -->
</head>
```

**推荐图片尺寸：**
| 平台 | 尺寸 | 比例 |
|------|------|------|
| Facebook | 1200×630 | 1.91:1 |
| Twitter | 1200×675 | 16:9 |
| LinkedIn | 1200×627 | 1.91:1 |
| 微信 | 900×500 | 16:9 |

#### 7. ARIA 无障碍访问

**问题**：ARIA 是什么？何时需要使用？

**完整答案**：

**ARIA（Accessible Rich Internet Applications）** 是一组属性，用于增强动态内容和自定义组件的无障碍访问能力，帮助屏幕阅读器理解页面。

**核心 ARIA 属性：**

```html
<!-- 1. role - 定义元素角色 -->
<div role="button">点击我</div>
<div role="navigation">导航区域</div>
<div role="main">主要内容区域</div>
<div role="complementary">辅助内容（侧边栏）</div>
<div role="contentinfo">页面信息（页脚）</div>
<div role="alert">重要提示信息</div>
<div role="dialog">对话框</div>
<div role="tablist">选项卡列表</div>
<div role="tab">选项卡</div>
<div role="tabpanel">选项卡面板</div>
<div role="menu">菜单</div>
<div role="menuitem">菜单项</div>

<!-- 2. aria-label - 提供无障碍标签（当没有可见标签时） -->
<button aria-label="关闭">×</button>
<input type="text" aria-label="搜索">
<nav aria-label="主导航">...</nav>

<!-- 3. aria-labelledby - 引用已有元素作为标签 -->
<span id="search-label">搜索</span>
<input type="text" aria-labelledby="search-label">

<!-- 4. aria-expanded - 标记展开/折叠状态 -->
<button aria-expanded="false" aria-controls="menu">菜单</button>
<ul id="menu" hidden>...</ul>

<!-- 5. aria-hidden - 隐藏元素（屏幕阅读器不可见） -->
<div aria-hidden="true">装饰性图标</div>

<!-- 6. aria-describedby - 引用描述文本 -->
<button aria-describedby="btn-desc">删除</button>
<p id="btn-desc">此操作不可撤销</p>

<!-- 7. aria-live - 动态内容更新通知 -->
<div aria-live="polite">  <!-- 礼貌：当前操作完成后朗读 -->
    状态消息会在此显示
</div>
<div aria-live="assertive">  <!-- 强制：立即中断并朗读 -->
    重要警告！
</div>

<!-- 8. aria-controls - 引用被控制的元素 -->
<button aria-expanded="false" aria-controls="dropdown">
    下拉菜单
</button>
<ul id="dropdown">...</ul>

<!-- 9. aria-current - 当前位置标记 -->
<nav>
    <a href="/" aria-current="page">首页</a>
    <a href="/about">关于</a>
</nav>

<!-- 10. aria-disabled - 禁用状态 -->
<button aria-disabled="true">不可用</button>
```

**什么时候需要使用 ARIA：**

| 场景 | 示例 | 推荐 ARIA |
|------|------|----------|
| 自定义组件 | 自制的下拉菜单、模态框 | role、aria-expanded、aria-controls |
| 图标按钮 | 只有图标没有文字 | aria-label |
| 动态内容更新 | 实时通知、加载状态 | aria-live |
| 复杂导航 | 树形菜单、选项卡 | role、aria-expanded |
| 表单错误提示 | 验证错误信息 | aria-describedby、aria-invalid |
| 数据表格 | 复杂数据展示 | role="grid"、aria-sort |

**ARIA 优先规则：**
```
1. 首选语义化 HTML 元素
2. 其次使用 ARIA 增强
3. 最后考虑自定义
```

```html
<!-- 错误：可以用原生就不要用 ARIA -->
<div role="button" tabindex="0" onclick="handleClick()">点击</div>

<!-- 正确：使用原生按钮 -->
<button onclick="handleClick()">点击</button>

<!-- 正确：必须用 ARIA 的场景 -->
<div class="custom-dropdown" role="button" aria-expanded="false" aria-haspopup="true">
    自定义下拉
</div>
```

**无障碍最佳实践：**
```html
<!-- 1. 图片必须有 alt -->
<img src="logo.png" alt="公司 Logo">
<img src="decorative.png" alt="" role="presentation">  <!-- 装饰性图片 -->

<!-- 2. 表单必须有 label -->
<label for="email">邮箱</label>
<input type="email" id="email" name="email">

<!-- 3. 跳过导航链接 -->
<a href="#main" class="skip-link">跳到主要内容</a>

<!-- 4. 颜色对比度足够（至少 4.5:1） -->

<!-- 5. 键盘可访问 -->
<div role="button" tabindex="0" onkeydown="handleKeydown(event)">
    可用 Tab 聚焦，Enter/Space 触发
</div>
```
