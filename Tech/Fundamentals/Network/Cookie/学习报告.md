## 🎯 核心要点总结

### 已掌握
- Cookie 本质是解决 HTTP 无状态问题
- 四种存储方案的选型策略（Cookie / localStorage / sessionStorage / Session）
- HttpOnly 防 XSS、Secure 防中间人、SameSite 防 CSRF 的分工
- Cookie 可见性规则（Domain + Path + Secure + HttpOnly 综合判断）
- CSRF 攻击原理与 SameSite 防御机制

### 待加强
- **Domain 子域共享**：不指定 = host-only；指定 `.example.com` = 所有子域
- **跨域三条件**：前端 `credentials: 'include'` + 服务端 CORS 双响应头 + Cookie `SameSite=None; Secure`
- **Cookie 前缀**：`__Host-`（最高安全，host-only）vs `__Secure-`（只需 Secure，可跨子域）
- **跨站 vs 跨域**：同站（eTLD+1 相同）比同源（协议+域名+端口）更宽松
