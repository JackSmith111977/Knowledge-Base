## 二、理解过程

### 2.1 初始理解

学习开始时，我对这些概念的初步认识：

> **JSX**：通过 JS 写 HTML 结构，需要 Babel 编译成虚拟 DOM
> **虚拟 DOM**：轻量化 JS 对象，避免频繁操作真实 DOM
> **Fiber**：解决递归栈更新无法暂停的问题
> **Hooks**：赋予函数组件生命周期，链表结构触发

### 2.2 发现的盲点

| 盲点 | 具体表现 | 如何解决 |
|------|----------|----------|
| JSX 本质 | 误以为是"JS 超集" | 理解为"披着 HTML 外衣的 JS" |
| Diff 时间复杂度 | 未理解为什么只同层比较 | O(n³) vs O(n)，100 万倍差异 |
| key 的作用 | 误以为 index 变化导致问题 | 列表顺序变化时，相同 index 对应不同数据 |
| Fiber 链表优势 | 只知道能找到父/子/兄弟 | 核心是支持迭代遍历，可中断恢复 |
| Hooks 优势 | 误以为类组件没有生命周期 | 类组件有生命周期，Hooks 优势是逻辑复用和代码组织 |

### 2.3 最终理解

经过查漏补缺后，我对这些概念的完整理解：

> **JSX**：就像"穿着西装的 JS"，看起来像 HTML，本质是 JS 函数调用，需要"翻译官"Babel 编译成浏览器能懂的 JS
>
> **虚拟 DOM**：就像"设计蓝图"，描述最终 UI 的结构，通过 Diff 算法计算最小变更，一次性批量应用，减少重排重绘
>
> **Fiber 架构**：就像"读书 + 书签"，每读一页（5ms）可以暂停，书签（Fiber 节点）保存进度，随时可以继续，让浏览器有空处理用户输入
>
> **Hooks**：就像"食谱"，规定各阶段做什么，必须按顺序执行（不能放在条件/循环中），用"记账本"存储状态

## 四、核心要点

本次学习的核心概念总结：

1. **JSX 语法**：JS 的语法扩展，编译成 `React.createElement()` 或 `_jsx()` 调用，返回虚拟 DOM 对象
2. **虚拟 DOM**：轻量级 JS 对象，Diff 算法三策略（同层比较 O(n)、类型判断 O(1)、key 优化）
3. **Fiber 架构**：链表结构替代树结构，支持可中断的异步渲染，5ms 时间片机制
4. **Hooks 原理**：内部维护状态链表，必须按顺序调用（不能在条件/循环中）

| 抽象概念 | 生活化比喻 | 说明 |
|----------|------------|------|
| JSX | "穿着西装的 JS" / "文言文" | 看起来像 HTML，本质是 JS，需要翻译 |
| JSX 编译 | "翻译官" | Babel 把 JSX 翻译成浏览器能懂的 JS |
| 虚拟 DOM | "设计蓝图" / "户口本" | 描述最终结构，轻量级表示 |
| Diff 批量更新 | "确认所有更改一起去" | 减少真实 DOM 操作次数 |
| Fiber | "读书 + 书签" / "做作业任务清单" | 可暂停、可恢复、知道进度 |
| Fiber 时间片 | "5ms 无感知" | 人眼感觉不到的短暂暂停 |
| Hooks | "食谱" / "考试计划" | 规定各阶段做什么，必须按顺序 |
| useState | "记分本" / "账本" | 存储状态，可查可改 |
| useEffect 依赖 | "触发条件清单" | 清单空=只一次，有依赖=变化时 |
