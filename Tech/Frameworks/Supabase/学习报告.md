## 核心要点总结

1. **五层架构**：客户端层 → Kong API 网关 → 核心服务（Auth/Realtime/Storage/Edge）→ PostgreSQL + S3
2. **PostgREST**：Haskell 编写，端口 3000，自动将数据库 Schema 映射为 RESTful API
3. **RLS 行级安全**：USING（读可见）+ WITH CHECK（写允许），默认拒绝所有访问
4. **双 Token 机制**：Access Token（JWT，1h）+ Refresh Token（Opaque，7d），轮换防窃取
5. **Realtime CDC**：Elixir 读取 PostgreSQL WAL 日志，通过 WebSocket 推送，延迟 ~100ms
6. **Storage 元数据分离**：文件在 S3，元数据在 PostgreSQL，支持 SQL 查询和 RLS 权限
7. **Edge Functions**：Deno 运行时，全球边缘部署，冷启动 <50ms，最长 60 秒
8. **防超卖**：PostgreSQL `SELECT ... FOR UPDATE` 行级锁，确保并发安全

## 记忆锚点

- **一库四服务一网关**：PostgreSQL + Auth/Realtime/Storage/Edge + Kong
- **双 Token 一短一长**：Access 1h / Refresh 7d，轮换防窃取
- **WAL 日志驱动 Realtime**：CDC 不是触发器，是预写日志复制
- **元数据在 PG 文件在 S3**：Storage 的核心差异化设计
- **前端 Anon 后端 Service**：永远不要在前端暴露 Service Role Key
