### 二、理解过程

#### 初始理解
学习者在使用 Next.js 前已有一定的 React 基础，对 SSR、SSG 等概念有初步了解。

#### 发现的盲点及解决

| 盲点 | 初始理解 | 正确理解 |
|------|----------|----------|
| **SSG vs ISR** | ISR 是"增量渲染" | ISR 是后台重新生成完整页面后替换 |
| **generateStaticParams** | "跳转时运行" | 构建时执行，预生成静态页面 |
| **error.tsx vs not-found.tsx** | 都是处理 404 | error 处理渲染错误，not-found 处理 404 |
| **HMR vs 刷新** | 都是重新加载 | HMR 保持组件状态，刷新丢失状态 |
| **SSG 部署 vs ISR 更新** | 混淆两者 | SSG 部署是原子替换，ISR 是后台更新 |
| **Server Actions 用途** | 可用于数据获取 | 专用于数据变更（Mutation） |
| **Edge Runtime vs 边缘节点** | 认为是同一概念 | Edge Runtime 是运行环境，边缘节点是物理位置 |

#### 最终理解
通过 8 章学习，学习者能够：
- 准确区分四种渲染模式及其适用场景
- 正确设计 Server/Client Component 的组件结构
- 理解 ISR 的 stale-while-revalidate 机制
- 识别并避免常见的水合错误和性能陷阱

### 四、核心要点总结

#### 1. 四种渲染模式

| 模式 | HTML 生成时机 | 适用场景 |
|------|--------------|----------|
| CSR | 浏览器端 | Dashboard、后台系统 |
| SSR | 每次请求 | 个性化内容、实时数据 |
| SSG | 构建时 | 博客、文档等静态内容 |
| ISR | 构建后按需更新 | 商品列表、新闻等半动态内容 |

#### 2. Server vs Client Component

| 维度 | Server | Client |
|------|--------|--------|
| 执行位置 | 服务端 | 客户端 |
| 数据库访问 | ✅ | ❌ |
| 浏览器 API | ❌ | ✅ |
| 交互能力 | ❌ | ✅ |
| 包体积 | 0 | 有 |
| 指令 | `'use server'` | `'use client'` |

**设计原则：** SC 读，SA 写，CC 负责交互行为

#### 3. 缓存策略

```javascript
// ISR 时间缓存
fetch(url, { next: { revalidate: 3600 } })

// 标签缓存（可批量失效）
fetch(url, { next: { tags: ['products'] } })
revalidateTag('products', 'max')

// 不缓存（实时数据）
fetch(url, { cache: 'no-store' })
```

#### 4. 水合错误避免

- 避免在服务端和客户端生成不同内容（时间戳、随机数）
- 使用 `useEffect` 延迟浏览器特定代码
- 暗黑模式使用内联脚本注入

### 五、待加强内容

1. **SSG vs ISR 的本质区别**
   - SSG：构建时生成，部署时原子替换
   - ISR：后台增量更新，首次请求返回旧内容

2. **HMR 的核心优势**
   - 保持组件状态（表单、计数器、滚动位置）
   - 与浏览器刷新的本质区别

3. **revalidateTag vs revalidatePath**
   - Tag：跨页面批量失效
   - Path：仅失效指定页面
