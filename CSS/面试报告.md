#### 1. CSS 引入方式

**问题**：CSS 有哪些引入方式？各有什么优缺点？

**完整答案**：

**三种标准引入方式：**

```html
<!-- 1. 外部样式表（推荐） -->
<link rel="stylesheet" href="styles.css">
<!-- 优点：样式与结构分离、可复用、浏览器缓存 -->
<!-- 缺点：需要额外 HTTP 请求 -->

<!-- 2. 内部样式表 -->
<style>
    p { color: blue; }
</style>
<!-- 优点：无需额外请求、适用于单页特定样式 -->
<!-- 缺点：不可复用、增加 HTML 文件大小 -->

<!-- 3. 行内样式（不推荐） -->
<div style="color: red;">内容</div>
<!-- 优点：优先级最高、适用于动态样式 -->
<!-- 缺点：不可复用、难以维护、违反分离原则 -->
```

**优先级顺序：**
```
行内样式 > 内部样式表 = 外部样式表（取决于源代码顺序）
```

**最佳实践：**
- 优先使用**外部样式表**，利于复用和维护
- 关键首屏样式可使用**内部样式表**减少渲染阻塞
- **行内样式**仅用于动态样式或邮件模板

**CSS Module 不是 CSS 标准引入方式：**
```javascript
// CSS Module 是构建工具（如 Webpack、Vite）提供的模块化方案
import styles from './Button.module.css';
// 编译后 class 名会被哈希处理，避免全局污染
```

#### 2. 层叠上下文 (Stacking Context)

**问题**：什么是层叠上下文？什么条件下会创建层叠上下文？

**完整答案**：

**层叠上下文**是一个三维概念，决定了元素在 Z 轴上的堆叠顺序。每个层叠上下文独立于其他上下文，子元素无法突破父元素的层级。

**创建层叠上下文的条件（满足任一即可）：**

```css
/* 1. 根元素 */
html { }  /* 自动创建 */

/* 2. position + z-index (z-index 不为 auto) */
.element {
    position: relative;  /* 或 absolute/fixed/sticky */
    z-index: 10;
}

/* 3. flex/grid 容器的子元素 */
.container {
    display: flex;  /* 或 grid */
}
.child {
    z-index: 10;  /* 子元素自动创建层叠上下文 */
}

/* 4. opacity < 1 */
.element {
    opacity: 0.9;
}

/* 5. transform 不为 none */
.element {
    transform: rotate(45deg);
}

/* 6. filter 不为 none */
.element {
    filter: blur(5px);
}

/* 7. will-change 准备创建 */
.element {
    will-change: transform;
}

/* 8. 包含渲染上下文 */
.element {
    contain: layout;
}
```

**层叠顺序规则：**

```
同一层叠上下文内的堆叠顺序（从底到顶）：
1. background/border（背景和边框）
2. 负 z-index 的子元素
3. 块级非定位子元素
4. 浮动子元素
5. 行内非定位子元素
6. z-index: auto 或 z-index: 0 的定位子元素
7. 正 z-index 的子元素（按 z-index 值排序）
```

**常见误区：**

```css
/* 错误：以为 z-index 可以跨层叠上下文比较 */
.parent1 {
    position: relative;
    z-index: 1;  /* 创建了层叠上下文 */
}
.child1 {
    position: relative;
    z-index: 999;  /* 只能在 .parent1 内比较 */
}

.parent2 {
    position: relative;
    z-index: 2;  /* .parent2 整体在 .parent1 之上 */
}
.child2 {
    position: relative;
    z-index: 1;  /* 即使 z-index 小，也在 .child1 之上 */
}
```

**调试技巧：**
```javascript
// 使用浏览器 DevTools 的 Layers 面板查看层叠上下文
// 或使用以下代码高亮层叠上下文元素
document.querySelectorAll('*').forEach(el => {
    const style = getComputedStyle(el);
    const createsContext =
        style.position !== 'static' && style.zIndex !== 'auto' ||
        style.opacity < 1 ||
        style.transform !== 'none' ||
        style.filter !== 'none' ||
        ['flex', 'grid'].includes(style.display);

    if (createsContext) {
        el.style.outline = '2px solid red';
    }
});
```

#### 3. CSS 变量（自定义属性）

**问题**：什么是 CSS 变量？如何定义和使用？

**完整答案**：

**CSS 变量（Custom Properties）** 允许将值存储在变量中，通过 `var()` 函数引用，实现样式的复用和动态修改。

**定义方式：**

```css
/* 全局变量（推荐在 :root 定义） */
:root {
    --primary-color: #3B82F6;
    --secondary-color: #64748B;
    --accent-color: #F59E0B;
    --spacing-sm: 0.5rem;
    --spacing-md: 1rem;
    --spacing-lg: 2rem;
    --font-size-base: 16px;
    --border-radius: 4px;
    --box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
}

/* 局部变量（作用域限制在该选择器内） */
.card {
    --card-bg: #fff;
    --card-border: #e2e8f0;
}
```

**使用方式：**

```css
/* 基本用法 */
.button {
    background-color: var(--primary-color);
    padding: var(--spacing-md);
    border-radius: var(--border-radius);
    box-shadow: var(--box-shadow);
}

/* 带后备值（当变量未定义时使用） */
.text {
    color: var(--primary-color, #3B82F6);
}

/* 变量嵌套 */
.component {
    --base-spacing: 1rem;
    padding: calc(var(--base-spacing) * 2);
}
```

**JavaScript 操作：**

```javascript
// 获取变量值
const root = document.documentElement;
const primaryColor = getComputedStyle(root)
    .getPropertyValue('--primary-color')
    .trim();

// 设置/修改变量（全局）
root.style.setProperty('--primary-color', '#EF4444');

// 设置/修改变量（局部）
const element = document.querySelector('.card');
element.style.setProperty('--card-bg', '#f0f0f0');

// 删除变量
root.style.removeProperty('--primary-color');
```

**实际应用场景：**

```css
/* 1. 主题切换 */
:root {
    --bg-color: #ffffff;
    --text-color: #1f2937;
}

[data-theme="dark"] {
    --bg-color: #1f2937;
    --text-color: #ffffff;
}

body {
    background-color: var(--bg-color);
    color: var(--text-color);
}

/* JavaScript 切换 */
document.documentElement.setAttribute('data-theme', 'dark');

/* 2. 响应式设计 */
:root {
    --font-size: 14px;
}

@media (min-width: 768px) {
    :root {
        --font-size: 16px;
    }
}

body {
    font-size: var(--font-size);
}

/* 3. 动态间距系统 */
.card {
    --gap: var(--spacing-md);
    display: flex;
    gap: var(--gap);
}

@media (max-width: 640px) {
    .card {
        --gap: var(--spacing-sm);
    }
}

/* 4. 组件状态管理 */
.button {
    --button-bg: var(--primary-color);
    --button-hover: var(--secondary-color);
    background-color: var(--button-bg);
}

.button:hover {
    --button-bg: var(--button-hover);
}

.button--secondary {
    --primary-color: var(--secondary-color);
    --secondary-color: var(--primary-color);
}
```

**与传统预处理器变量的区别：**

| 特性 | CSS 变量 | Sass/Less 变量 |
|------|----------|---------------|
| 定义符号 | `--variable` | `$variable` |
| 引用方式 | `var(--variable)` | `$variable` |
| 运行时修改 | ✅ 支持（JavaScript） | ❌ 编译时替换 |
| 作用域 | 层叠作用域 | 文件/块级作用域 |
| 继承性 | ✅ 支持 | ❌ 不支持 |

**浏览器支持：**
- Chrome 49+
- Firefox 31+
- Safari 9.1+
- Edge 15+
- 不支持 IE
