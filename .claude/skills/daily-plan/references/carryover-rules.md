# 任务结转规则

> 定义未完成任务如何在日与日之间流转、降级、归档。

---

## 任务状态定义

| 状态 | 标记 | 含义 |
|------|------|------|
| 已完成 | `[x]` | 任务已完成，不结转 |
| 进行中 | `[-]` | 已开始但未完成，优先结转 |
| 未开始 | `[ ]` | 未开始，根据优先级结转 |
| 已延期 | `[>]` | 用户主动延后到指定日期 |

---

## 结转规则

### 1. 读取昨日索引

从 `Daily/PROGRESS.md` 中读取最近一天的条目：
- 查找"状态"不是"已完成"的最近一条记录
- 提取其"未完成项"列表

### 2. 自动结转条件

| 任务年龄 | 操作 | 说明 |
|----------|------|------|
| 1 天未完成 | 原优先级结转 | 直接进入今日计划 |
| 2 天未完成 | 提升优先级 | 标记为"需关注" |
| ≥ 3 天未完成 | 降低优先级 | 标记为"长期未决"，放入最低优先级时间段 |

### 3. 用户确认环节

结转前向用户展示：
```
以下 N 个未完成任务将结转至今日计划：
1. [来源日期] 任务名（已 X 天）→ 优先级: 高/中/低
请确认：保留全部 / 选择保留 / 全部删除
```

### 4. 索引更新

结转确认后，在 `Daily/PROGRESS.md` 中追加新条目：

```markdown
## YYYY-MM-DD
- 状态：进行中
- 重点：XX + XX
- 完成项：[]
- 未完成项：[]
- 文件：Daily/YYYY-MM-DD.md
```

---

## PROGRESS.md 完整格式

```markdown
# 每日任务索引

> 用于快速读取历史任务概览，不写入完整计划内容

---

## YYYY-MM-DD
- 状态：已完成/进行中/未开始
- 重点：复习方向1 + 复习方向2
- 完成项：[任务A, 任务B]
- 未完成项：[任务C → 结转至次日]
- 算法：[滑动窗口 x3 / 二分 x2]
- 文件：Daily/YYYY-MM-DD.md

---
```

---

## 过期处理

- 超过 **7 天** 未完成的单项任务：标记为"已过期"，从结转列表移除
- 用户可随时手动恢复过期任务
- 过期的任务仍在对应的 Daily 文件中可追溯

---

## 与 progress.txt 的关系

- `progress.txt`：记录知识库文档的调研/创建进度（长期）
- `Daily/PROGRESS.md`：记录每日任务的执行进度（短期）
- 两者独立，不互相覆盖
- 当 `Daily/PROGRESS.md` 中的任务涉及文档创建/更新时，完成后需同步更新 `progress.txt`
