# 使用示例

> project-migration Skill 的使用示例

---

## 示例 1：单应用项目迁移

**用户：** `/project-migration` 或 "老项目接入 AI"

**响应流程：**

```
🚀 项目迁移向导
→ 阶段 0：工作区检测（单应用）
→ 阶段 1：现状分析 → current-status.md
→ 阶段 2：无限询问（动态追问）
→ 阶段 3：文档生成 → 完成报告
```

**详细交互：**

1. 用户触发 Skill
2. AI 检测工作区类型（单应用）
3. AI 扫描项目，生成现状报告
4. AI 询问项目核心目标、团队配置、文档详细程度等
5. AI 生成完整文档体系
6. 输出完成报告

---

## 示例 2：Trae 工作区多应用迁移

**用户：** "分析这个项目"

**响应流程：**

```
🚀 项目迁移向导
→ 阶段 0：工作区检测（Trae 工作区，2 个应用）
→ 检测 .trae/ 配置文件
→ 列出配置的应用根目录
→ 询问是否分析所有应用或手动添加
→ 阶段 1：现状分析 → current-status.md（包含两个应用）
→ 阶段 2：无限询问（增加多应用配置问题）
→ 用户选择：混合模式文档
→ 阶段 3：文档生成 → 完成报告
```

**详细交互：**

1. 用户触发 Skill
2. AI 检测到 Trae 工作区配置
3. AI 列出配置的应用根目录
4. 用户确认分析所有应用
5. AI 扫描两个应用，生成整体现状报告
6. AI 追问多应用文档组织方式
7. 用户选择混合模式
8. AI 生成文档体系
9. 输出完成报告

## 示例 3：手动指定另一个应用根目录

**用户：** "迁移到文档优先"

**响应流程：**

```
🚀 项目迁移向导
→ 阶段 0：工作区检测（检测到当前应用）
→ 用户选择：手动添加另一个应用的根目录地址
→ 用户提供：D:/WorkPlace/App2
→ 阶段 1：现状分析（扫描两个应用）→ current-status.md
→ 阶段 2：无限询问
→ 阶段 3：文档生成 → 完成报告
```

**详细交互：**

1. 用户触发 Skill
2. AI 检测当前工作区
3. AI 提供选项：分析所有应用 / 选择特定应用 / 手动添加
4. 用户选择手动添加，提供路径 `D:/WorkPlace/App2`
5. AI 扫描两个应用
6. 执行后续流程

## 示例 4：有现有文档的项目

**用户：** "为这个项目加文档"

**响应流程：**

```
🚀 项目迁移向导
→ 阶段 0：工作区检测
→ 阶段 1：现状分析（检测到部分文档）
→ AI 追问：现有文档是否需要保留
→ 阶段 2：无限询问
→ 阶段 3：文档生成（保留 + 补充）
```

**详细交互：**

1. 用户触发 Skill
2. AI 检测到项目已有 README 和部分设计文档
3. AI 在现状报告中标注文档质量评分
4. AI 追问哪些文档需要保留/更新
5. 用户确认
6. AI 生成文档体系（保留有用内容，补充缺失部分）
