# 多应用项目处理指南

> 本文档描述如何识别和处理 Monorepo、Trae、VSCode 工作区等多应用项目

---

## 工作区识别逻辑

| 工作区类型 | 识别标志 | 处理方式 |
|------------|----------|----------|
| **Trae 工作区** | `.trae/` 目录或工作区配置文件 | 读取配置中的应用根目录 |
| **VSCode 工作区** | `.code-workspace` 文件 | 解析 folders 数组 |
| **Monorepo** | `apps/`、`packages/` 目录 | 扫描子目录识别独立应用 |
| **单应用** | 标准项目结构 | 按单应用处理 |

---

## Trae 工作区处理

### 识别方法

检测 `.trae/` 目录或工作区配置文件。

### 处理流程

1. 读取 `.trae/` 配置文件
2. 解析配置的应用根目录列表
3. 逐个扫描每个应用
4. 生成整体现状报告

### 输出示例

```markdown
## 工作区检测结果

**类型：** Trae 工作区
**应用数量：** 2

**发现的应用：**
- admin-web：`apps/admin-web/`
- customer-app：`apps/customer-app/`
```

---

## VSCode 工作区处理

### 识别方法

检测 `.code-workspace` 文件。

### 处理流程

1. 解析 `.code-workspace` 文件中的 `folders` 数组
2. 提取每个文件夹的路径
3. 逐个扫描每个应用
4. 生成整体现状报告

---

## Monorepo 处理

### 识别方法

检测 `apps/`、`packages/` 目录。

### 处理流程

1. 扫描 `apps/` 目录下所有子目录
2. 识别每个子目录是否为独立应用（检查 package.json）
3. 扫描 `packages/` 目录识别共享包
4. 分析依赖关系
5. 生成整体现状报告

### 依赖关系分析

```
admin-web ──┬──> ui-components
            ├──> utils
            └──> api-client

customer-app ──┬──> ui-components
               ├──> utils
               └──> api-client
```

## 多应用文档策略

根据用户选择（统一/分离/混合）：

### 统一模式

所有应用共享一套文档。

```
docs/
└── main/
    ├── TECH_STACK.md（包含所有应用的技术栈）
    └── APP_FLOW.md（包含所有应用的流程）
```

**适用场景：** 应用间高度耦合、共享代码多

### 分离模式

每个应用独立文档。

```
docs/
├── apps/
│   ├── admin-web/
│   │   ├── TECH_STACK.md
│   │   └── APP_FLOW.md
│   └── customer-app/
│       ├── TECH_STACK.md
│       └── APP_FLOW.md
└── shared/
    └── TECH_STACK.md（共享包技术栈）
```

**适用场景：** 应用相对独立、团队分工明确

### 混合模式

核心共享文档 + 应用独立文档。

```
docs/
├── main/（核心共享文档）
│   ├── CLAUDE.md
│   ├── TECH_STACK.md（整体技术栈）
│   └── APP_FLOW.md（整体流程）
└── apps/（应用独立文档）
    ├── admin-web/
    │   ├── COMPONENTS.md
    │   └── PAGES.md
    └── customer-app/
        ├── FEATURES.md
        └── SCREENS.md
```

**适用场景：** 大部分共享 + 部分独立功能

## 询问用户配置

在阶段 2（无限询问）中，如检测到多应用，追问以下问题：

### 多应用文档组织方式

```markdown
### 多应用配置

检测到项目包含多个应用：
- [应用 1]
- [应用 2]

**文档组织方式：**
- [ ] 统一模式（所有应用共享一套文档）
- [ ] 分离模式（每个应用独立文档）
- [ ] 混合模式（核心共享 + 应用独立）

请选择或说明偏好。
```

### 应用间依赖管理策略

```markdown
### 应用间依赖管理

**当前依赖关系：**
[依赖关系图]

**依赖管理策略：**
- [ ] 保持现状
- [ ] 抽取共享包
- [ ] 重构循环依赖

请选择或说明偏好。
```

---

## 注意事项

### 循环依赖检测

如检测到应用间存在循环依赖，在现状报告中标注为风险点。

### 共享包处理

共享包的技术栈在共享文档中描述，应用特有的技术栈在各自文档中描述。

### 版本一致性

检查多个应用使用的框架版本是否一致，如不一致在现状报告中标注。
