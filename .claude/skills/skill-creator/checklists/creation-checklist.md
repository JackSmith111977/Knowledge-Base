# Skill 创建检查清单

> 在写入 Skill 文件前必须完成此检查

**版本：** 1.0.0 | **更新：** 2026-03-30

---

## 一、创建前检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 明确 2-3 个具体用例 | 有用例描述和预期结果 | - [ ] |
| 确认触发方式 | 显式命令 + 隐式触发词 | - [ ] |
| 选择 Skill 类型 | 9 种类型之一 | - [ ] |
| 确认文件结构需求 | templates/references/scripts/checklists | - [ ] |

---

## 二、生成后检查

### 格式检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| YAML Frontmatter 正确 | name/description/version 等字段 | - [ ] |
| description 清晰具体 | 100 词内，非笼统描述 | - [ ] |
| Markdown 格式正确 | 代码块、表格、列表规范 | - [ ] |

### 内容检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 包含可执行工作流程 | 具体步骤而非空泛描述 | - [ ] |
| 包含输出格式规范 | 明确输出什么 | - [ ] |
| 包含注意事项 | 边界情况、常见错误 | - [ ] |
| 没有陈述显而易见的内容 | 避免废话 | - [ ] |

### 触发检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 配置显式触发命令 | 如 `/skill-name` | - [ ] |
| 配置隐式触发词 | 3-5 个中文触发词 | - [ ] |

---

## 三、写入前检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 用户已确认草稿 | 变更摘要已确认 | - [ ] |
| Review 检查通过 | 本清单所有必须项通过 | - [ ] |
| 输出路径已确认 | 默认 `.claude/skills/[name]/` | - [ ] |

---

## 四、9 种 Skill 类型核对

| 类型 | 适用场景 | 是否匹配 |
|------|----------|----------|
| 库和 API 参考 | 内部库、CLI、SDK 使用指南 | - [ ] |
| 产品验证 | 测试流程、断言验证 | - [ ] |
| 数据获取与分析 | 数据查询、报表生成 | - [ ] |
| 代码生成 | 规格转代码、设计稿转前端 | - [ ] |
| 文档生成 | PR 描述、API 文档、发布说明 | - [ ] |
| 代码审查 | 安全检查、性能分析、代码质量 | - [ ] |
| 部署流程 | 部署、发布、回滚 | - [ ] |
| 团队规范 | 编码风格、提交规范 | - [ ] |
| 调研整理 | 技术调研、知识整理 | - [ ] |

---

## 五、安全检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 无敏感权限 | 不读取 .env 等敏感文件 | - [ ] |
| 脚本执行需确认 | 执行脚本前用户确认 | - [ ] |
| 外部 API 验证 SSL | 外部调用验证安全性 | - [ ] |

---

## 六、评估体系检查（Stage 9）

### Eval 定义检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| evals.json 存在 | `[skill-name]/evals/evals.json` | - [ ] |
| 至少 3 个测试用例 | 正常路径 + 边界 + 错误 | - [ ] |
| 每条期望可验证 | 非"质量好"类模糊描述 | - [ ] |
| 期望可区分 | 对明显错误输出不应通过 | - [ ] |

### Grader 准备检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| `references/grader-agent.md` 存在 | 评分器指令 | - [ ] |
| `references/eval-schemas.md` 存在 | 数据格式定义 | - [ ] |
| `references/eval-iteration-loop.md` 存在 | 迭代流程说明 | - [ ] |

### 版本追踪检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| `evals/history.json` 结构正确 | 包含 skill_name、iterations | - [ ] |
| 版本与 SKILL.md frontmatter 一致 | version 字段对应 | - [ ] |

---

## 必须项（不满足不能发布）

- [ ] YAML Frontmatter 格式正确
- [ ] description 清晰具体（100 词内）
- [ ] 包含可执行的工作流程
- [ ] 配置了显式触发命令
- [ ] 配置了隐式触发词（3-5 个）
- [ ] 没有陈述显而易见的内容
- [ ] 用户已确认草稿
