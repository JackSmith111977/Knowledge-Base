# Skill Creator 使用示例

> 展示 skill-creator Skill 的典型使用场景

**版本：** 1.0.0 | **更新：** 2026-03-30

---

## 示例 1：创建代码审查 Skill

**用户输入：**
```
我想创建一个 Skill，用于审查代码的安全性
```

**Skill 响应：**
```
📋 已收到 Skill 创建请求

根据描述，我推荐使用：**类型 6 - 代码审查**

## 需求澄清

请回答以下问题：

1. 主要审查哪方面的安全问题？
   （SQL 注入、XSS、认证授权、数据安全等）

2. 目标代码是什么语言/框架？

3. 是否需要执行静态分析工具（如 Semgrep）？

4. 输出格式偏好？
   （Markdown 报告、JSON 数据等）

5. 触发方式偏好？
   - 显式：/security-review
   - 隐式："检查安全问题"、"审计代码"
```

## 示例 2：优化已有 Skill

**用户输入：**
```
优化 research skill，精简主文档
```

**Skill 响应：**
```
📋 已收到 Skill 优化请求

## 当前状态分析

**主文档行数：** 580 行（超过建议的 500 行）
**问题：**
- 内容过长，Token 消耗大
- 缺少 references/目录结构
- 踩坑清单可以更详细

## 优化方案

1. 精简主文档至 ~250 行
2. 创建 references/目录，存放：
   - doc-structure.md（文档结构模板）
   - citation-guide.md（引用格式规范）
   - gotchas.md（踩坑清单）
3. 创建 checklists/目录
4. 创建 examples/目录

请确认优化方案。
```

## 示例 3：修改已有 Skill

**用户输入：**
```
修改 research Skill，添加更多触发词
```

**Skill 响应：**
```
📋 已收到 Skill 修改请求

## 读取 Skill

正在读取：`.claude/skills/research/SKILL.md`

...

## 当前触发词配置

```yaml
triggers: [调研，研究，整理一份，帮我调研，生成文档]
```

## 修改建议

建议新增以下触发词：
- "深度分析"
- "整理资料"
- "做调研"

请确认是否添加，或提供其他触发词。
```

## 示例 4：创建调研 Skill

**用户输入：**
```
帮我创建一个 Skill，用于调研新技术并生成知识文档
```

**Skill 响应：**
```
📋 已收到 Skill 创建请求

根据描述，我推荐使用：**类型 9 - 调研整理**

## 需求澄清

请回答以下问题：

1. 主要调研哪些类型的主题？
   （技术框架、业务方案、AI 工具等）

2. 是否需要网络搜索能力？
   - 是（配置 WebSearch/WebFetch）
   - 否（仅整理已有资料）

3. 输出文档的存储位置？
   - Knowledge Base/
   - 项目 docs/
   - 其他

4. 是否需要生成进度追踪？
   - 是（创建 progress.txt）
   - 否

5. 触发方式偏好？
   - 显式：/research
   - 隐式："调研"、"整理资料"
```

## 示例 5：创建部署流程 Skill

**用户输入：**
```
想创建一个 Skill，自动化部署流程到生产环境
```

**Skill 响应：**
```
📋 已收到 Skill 创建请求

根据描述，我推荐使用：**类型 7 - 部署流程**

## 需求澄清

请回答以下问题：

1. 部署目标环境是什么？
   （云服务器、容器平台、Serverless 等）

2. 当前使用的 CI/CD 工具？
   （GitHub Actions、Jenkins、GitLab CI 等）

3. 是否需要执行脚本？
   - 是（请提供脚本类型：Shell/Python/其他）
   - 否

4. 部署流程包含哪些步骤？
   （构建、测试、推送、部署等）

5. 是否需要回滚机制？
   - 是
   - 否
```
