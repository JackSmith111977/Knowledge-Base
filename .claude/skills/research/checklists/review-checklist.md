# Review 检查清单

> 在文档写入前必须完成此检查

---

## 一、结构检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 章节编号连续 | 1-8 编号无跳跃 | - [ ] |
| 无重复章节 | 没有重复的章节标题 | - [ ] |
| 目录对应 | 目录与实际章节一致 | - [ ] |
| 章节顺序合理 | 子章节在父章节之后 | - [ ] |

### 检查命令

```bash
# 检查章节编号
grep "^## " filename.md

# 检查重复标题
grep "^## " filename.md | sort | uniq -d
```

## 三、格式检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| Markdown 正确 | 代码块、表格、列表规范 | - [ ] |
| 分隔符统一 | 使用 `---` 分隔章节 | - [ ] |
| 代码块标注 | 标注语言（```javascript） | - [ ] |
| 表格完整 | 有表头分隔线 | - [ ] |
| **Mermaid 图表** | **使用 Mermaid 而非 ASCII 图** | - [ ] |

### 常见格式问题

```markdown
<!-- ❌ 错误：代码块无语言 -->
```
const x = 1;
```

<!-- ✅ 正确：标注语言 -->
```javascript
const x = 1;
```

<!-- ❌ 错误：表格无分隔线 -->
| 头 1 | 头 2 |
|------|------|
| 内容 | 内容 |

<!-- ✅ 正确：有分隔线 -->
| 头 1 | 头 2 |
|------|------|
| 内容 | 内容 |
```

### Mermaid 图表检查

```markdown
<!-- ❌ 错误：ASCII 流程图 -->
```
┌─────────────┐     ┌──────────────┐
│  State 变化  │ ──> │ 创建新 VDOM  │
└─────────────┘     └──────────────┘
```

<!-- ✅ 正确：Mermaid 流程图 -->
```mermaid
flowchart LR
    A[State 变化] --> B[创建新 VDOM]
    B --> C[Diff 对比]
```
```

**检查项：**
- [ ] 无 ASCII 字符绘制的流程图/架构图
- [ ] 流程图/架构图使用 `flowchart` 语法
- [ ] 状态图使用 `stateDiagram-v2` 语法
- [ ] 交互图使用 `sequenceDiagram` 语法
- [ ] 图表语法正确（可在 mermaid.live 验证）

## 问题修复指南

| 问题 | 修复方式 |
|------|----------|
| 章节编号不连续 | 重新编号 |
| 目录与实际不符 | 同步目录 |
| 代码块无语言标注 | 添加语言 |
| 引用格式不统一 | 统一格式 |
| 内容过浅 | 补充原理讲解 |
| **使用 ASCII 流程图** | **改造为 Mermaid** |
