## 模板详细特征

### T1：核心知识体系（默认模板）

- **Diátaxis 映射：** Tutorial + Explanation 混合
- **适用场景：** 通用技术/框架/库调研，用户需要全面系统地了解某项技术
- **判断特征：** 主题宽泛（如"React 核心知识体系"、"Next.js 全面解析"）
- **章节结构：** 概述 → 核心概念 → 快速入门 → 基础用法 → 高级特性 → 实战案例 → 常见问题 → 学习资源
- **典型用例：** 新框架学习、技术栈全景了解

### T2：知识图谱

- **Diátaxis 映射：** Explanation（概念网络）
- **适用场景：** 概念之间的实体关系建模，知识网络可视化
- **判断特征：** 主题涉及多个关联概念、实体、模式之间的关系（如"前端设计模式图谱"、"React 生态关系图"）
- **章节结构：** 图谱概览 → 实体定义 → 关系模型 → 属性映射 → 可视化图谱 → 应用场景 → 扩展阅读
- **典型用例：** 概念梳理、生态全景、知识网络

### T3：操作指南

- **Diátaxis 映射：** How-to Guide
- **适用场景：** 分步骤的操作流程、配置指南、部署手册
- **判断特征：** 包含明确的动作和步骤（如"Docker 容器部署指南"、"CI/CD 配置流程"）
- **章节结构：** 前置条件 → 环境准备 → 操作步骤（按序编号） → 验证方法 → 故障排除 → 常见问题
- **典型用例：** 安装部署、配置流程、运维操作

### T4：深度解析

- **Diátaxis 映射：** Explanation（深层）
- **适用场景：** 源码级分析、底层机制研究、架构设计拆解
- **判断特征：** 主题强调深入理解（如"React Fiber 架构源码解析"、"Vue 响应式原理深度拆解"）
- **章节结构：** 架构总览 → 核心数据结构 → 关键流程拆解 → 源码级分析 → 性能分析 → 扩展机制 → 设计哲学
- **典型用例：** 源码阅读、底层机制、架构设计

### T5：快速入门

- **Diátaxis 映射：** Tutorial（精简）
- **适用场景：** 5 分钟上手、备忘录、速查表
- **判断特征：** 强调速度和简洁（如"Tailwind CSS 速查表"、"Git 常用命令备忘"）
- **章节结构：** 环境要求 → 安装（1 条命令） → Hello World → 核心 API 速览 → 常见坑 → 扩展阅读
- **典型用例：** 快速上手、日常速查、备忘参考

### T6：API 参考

- **Diátaxis 映射：** Reference
- **适用场景：** 接口文档、SDK 参考、方法列表
- **判断特征：** 主题是 API/SDK/库的参考文档（如"Express.js API 参考"、"REST API 接口文档"）
- **章节结构：** 概述 → 认证 → 端点列表 → 请求/响应格式 → 参数详解 → 错误码 → 速率限制 → 示例代码
- **典型用例：** SDK 文档、接口参考、方法手册

### T7：概念解析

- **Diátaxis 映射：** Explanation（纯理论）
- **适用场景：** 纯理论、设计思想、范式研究
- **判断特征：** 主题抽象、理论性强（如"依赖注入范式"、"函数式编程思想"）
- **章节结构：** 定义与起源 → 核心原则 → 演进历史 → 范式对比 → 应用场景 → 反模式 → 延伸阅读
- **典型用例：** 编程范式、设计思想、架构理论

### T8：最佳实践

- **Diátaxis 映射：** How-to + Reference 混合
- **适用场景：** 生产经验总结、反模式清单、检查列表
- **判断特征：** 主题强调实战经验（如"React 生产环境最佳实践"、"Node.js 性能优化指南"）
- **章节结构：** 场景分类 → 实践清单 → 反模式（❌ → ✅） → 检查列表 → 性能基准 → 工具推荐
- **典型用例：** 生产经验、性能优化、安全加固

### T9：对比分析

- **Diátaxis 映射：** Explanation + Reference 混合
- **适用场景：** 技术选型、方案对比、迁移指南
- **判断特征：** 主题涉及两个或多个方案的比较（如"React vs Vue vs Angular"、"REST 迁移到 GraphQL"）
- **章节结构：** 对比对象 → 维度矩阵 → 场景匹配 → 迁移指南 → 决策树 → 推荐方案
- **典型用例：** 技术选型、方案对比、迁移决策

## 混合模板场景

当调研主题跨越多种模板类型时，使用以下策略：

### 策略 1：主模板 + 子章节

- 确定一个主模板类型（覆盖 60%+ 内容）
- 其他模板类型作为子章节嵌入
- **示例：** "React 核心知识体系"（T1 为主）+ 嵌入 T5（快速入门章节）+ 嵌入 T8（最佳实践章节）

### 策略 2：系列文档

- 将大主题拆分为多个模板类型，每个生成独立文档
- 通过交叉引用链接关联
- **示例：** "Vue.js 全面调研" → T5（快速入门）+ T1（核心知识）+ T4（深度解析）+ T8（最佳实践）

### 策略 3：用户指定

- 如果用户明确要求特定模板类型，优先遵循用户意愿
- 阶段 2 输出时推荐主模板，同时列出备选方案供用户选择
