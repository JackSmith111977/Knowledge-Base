## 3. 关系模型

### 3.1 关系类型

| 关系 | 源实体 | 目标实体 | 基数 | 语义说明 |
|------|--------|----------|------|---------|
| [关系名] | [A] | [B] | 1:N / M:N / 1:1 | A 和 B 之间是什么关系 |

### 3.2 ER 图

```mermaid
erDiagram
    ENTITY_A ||--o{ ENTITY_B : "关系描述"
    ENTITY_A ||--|| ENTITY_C : "关系描述"
    ENTITY_B }o--o{ ENTITY_D : "关系描述"
```

**编写指引：**
- 基数符号：`||` 一对一、`|{` 一对多、`}{` 多对多、`||--o{` 一对零到多
- 每个关系必须有语义说明，解释"为什么存在这个关系"
- 先列直接关系，再列间接/派生关系

## 5. 可视化图谱

### 5.1 概念关系图

```mermaid
graph TD
    A[核心概念] --> B[子概念1]
    A --> C[子概念2]
    B --> D[衍生概念]
    C --> D
    B -. 可选关联 .-> E[外部概念]
    C --> F[相关概念]
```

### 5.2 思维导图

```mermaid
mindmap
  root((核心主题))
    分支1
      子项1
      子项2
    分支2
      子项3
      子项4
    分支3
      子项5
```

**编写指引：**
- `graph TD` 适合展示因果链、流程依赖、层次结构
- `mindmap` 适合快速浏览概念全貌
- `erDiagram` 适合精确的实体关系建模
- 节点命名要简洁，详细解释放在正文中
