# 闪卡生成模板

**版本：** 1.0.0
**最后更新：** 2026-04-14

---

## 用途

定义闪卡模式下，AI 如何基于文档原文生成 6 段式闪卡的模板。

---

## 闪卡 6 段式结构

### 1. 📌 核心概念（≤ 30 字）

**规则：**
- 必须是一句话定义
- 不能是代码片段
- 必须覆盖该小节最核心的概念
- 不超过 30 个中文字

**示例：**
```
📌 核心概念
useState 是 React 函数组件中用于声明状态和状态更新函数的 Hook。
```

### 2. 🔑 关键要点（3-5 条）

**规则：**
- 每条不超过 20 字
- 必须覆盖核心 API 用法、核心规则、注意事项
- 按重要性排序
- 3 条是底线，最多 5 条

**示例：**
```
🔑 关键要点
- 调用 useState(initialValue) 返回 [state, setState] 数组
- setState 触发组件重新渲染，状态更新是异步的
- 多次调用 setState 会被 React 批量合并
- 状态更新函数不需要手动绑定 this
- 只能在组件顶层调用，不能在条件/循环/嵌套函数中调用
```

### 3. ⚠️ 常见误区（1-2 条）

**规则：**
- 必须是开发者真正容易犯错的点
- 格式：「误区描述 → 正确理解」
- 优先选择初学者高频误区

**示例：**
```
⚠️ 常见误区
- 误区：setState 后立即读取 state 能拿到新值 → 正确：state 更新是异步的，当前渲染周期内 state 仍是旧值
- 误区：可以用 if 条件调用 useState → 正确：Hooks 必须按固定顺序调用，不能在条件语句中使用
```

### 4. 💡 记忆口诀（1 句话）

**规则：**
- 可以是口诀、比喻、谐音、联想
- 一句话完成
- 要有趣且好记

**示例：**
```
💡 记忆口诀
「一调两值」：一次调用 useState，返回当前状态和更新函数两个值。
```

### 5. 🔗 关联知识

**规则：**
- 列出 2-3 个与当前知识直接关联的其他知识点
- 简述关系（不超过 15 字）
- 优先关联已学过的知识

**示例：**
```
🔗 关联知识
- 与 useEffect 的关系：state 变化会触发依赖该 state 的 useEffect 回调
- 与 useMemo 的关系：复杂 state 计算可用 useMemo 避免重复计算
```

### 6. ❓ 自测题（1 道）

**规则：**
- 优先判断题或选择题
- 选项必须有迷惑性
- 答案必须在闪卡覆盖的知识点范围内

**判断题格式：**
```
❓ 快速自测

以下说法是否正确？

"React 中调用 useState 返回的 setState 函数后，当前渲染周期内可以立即读取到新的 state 值。"

回复「正确」或「错误」，或回复：
- 「✅ 掌握」→ 进入下一张闪卡
- 「⚠️ 模糊」→ 展开详细解释
- 「❌ 忘了」→ 展开详细解释 + 加入错题本
```

**选择题格式：**
```
❓ 快速自测

以下关于 useState 的说法，哪项是正确的？

A. useState 可以在 if 条件语句中按需调用
B. setState 是同步的，调用后立即读取到新值
C. useState 返回的数组第一项是状态，第二项是更新函数
D. 多个 setState 调用不会被合并，每个都会触发重新渲染

回复 A/B/C/D，或回复：
- 「✅ 掌握」→ 进入下一张闪卡
- 「⚠️ 模糊」→ 展开详细解释
- 「❌ 忘了」→ 展开详细解释 + 加入错题本
```

---

## 闪卡生成约束

| 约束 | 规则 |
|------|------|
| **来源约束** | 每张闪卡内容必须来自文档原文，不准编造 |
| **数量约束** | 每个小节生成 1 张闪卡 |
| **长度约束** | 整张闪卡不超过 300 字（不含自测题选项） |
| **出题约束** | 自测题必须覆盖关键要点中的至少一个 |

---

## 闪卡质量检查（AI 内部执行）

生成闪卡后，AI 必须内部验证：

| 检查项 | 规则 |
|--------|------|
| 核心概念长度 | 不超过 30 字 |
| 关键要点数量 | 3-5 条 |
| 常见误区数量 | 1-2 条 |
| 记忆口诀 | 必须存在 |
| 关联知识 | 至少 1 条 |
| 自测题答案 | 必须有明确答案 |
