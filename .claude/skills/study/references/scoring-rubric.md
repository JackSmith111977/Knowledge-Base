# 评分标准与反馈规范

**用途：** 为学习者的每个回答提供一致、可追踪的评分

---

## 评分等级（0-5 分）

| 分数 | 等级 | 标准描述 |
|------|------|----------|
| **5** | 优秀 | 准确完整，能用自己的话解释，无知识盲点，能举一反三 |
| **4** | 良好 | 基本正确，有小遗漏或表述不够清晰，核心概念无误 |
| **3** | 合格 | 核心概念正确，有明显遗漏或误解，需要补充 |
| **2** | 待改进 | 部分正确，但关键概念有错误，需要重点讲解 |
| **1** | 不足 | 回答与问题关联度低，理解偏差较大 |
| **0** | 未答 | 未回答或完全错误 |

---

## 反馈格式模板

```markdown
### 评分：[X]/5 [等级]

**✅ 正确部分：**
- [具体正确点 1]
- [具体正确点 2]

**⚠️ 需要补充/纠正：**
- [遗漏或错误点 1]
- [正确理解应该是...]

**📌 下一步：**
- [追问/下一题/进入下一环节]
```

---

## 追问策略

### 根据评分决定追问

| 评分 | 策略 |
|------|------|
| **5 分** | 直接下一题，或增加难度追问 |
| **4 分** | 简单追问确认理解 |
| **3 分** | 补充讲解后，换角度再问 |
| **2 分及以下** | 详细讲解 + 示例，然后重新测试 |

### 追问示例

**用户回答评分 4 分：**
```
评分：4/5 良好

✅ 正确部分：SSR 在服务端生成 HTML 理解正确

⚠️ 需要补充：没提到 SEO 优势

📌 追问：SSR 相比 CSR 对 SEO 有什么帮助？为什么？
```

**用户回答评分 2 分：**
```
评分：2/5 待改进

⚠️ 误解点：你认为 SSG 是每次请求生成页面，这是错误的

📌 讲解：SSG 是在构建时生成一次 HTML...

📌 重新测试：请用一句话概括 SSG 的核心特点
```

## 评分记录格式

学习报告中记录每题评分：

```markdown
## 测验评分记录

| 题号 | 问题 | 评分 | 等级 | 知识点 |
|------|------|------|------|--------|
| 1 | 什么是 SSR？ | 4/5 | 良好 | 渲染模式 |
| 2 | SSR vs CSR 区别？ | 5/5 | 优秀 | 渲染模式 |
| 3 | ISR 的工作原理？ | 3/5 | 合格 | 缓存策略 |

**平均分：** 4.0/5
**掌握度评估：** 良好，需加强 ISR 理解
```

---

## 评分一致性检查

**跨题目评分应保持一致标准：**

- 同一知识点的不同问题，评分标准应统一
- 避免前松后紧或前紧后松
- 记录典型回答作为评分参考样本
