# 闪卡复习模式流程模板（Flashcard Review Mode）

**版本：** 1.0.0
**最后更新：** 2026-04-14

**适用场景：** 考前冲刺 / 知识唤醒 / 通勤复习
**预计耗时：** 3-5 分钟/章
**设计模式：** Pipeline + Inversion

---

## 流程概览

```
Step 1：加载章节 → Step 2：生成闪卡 → Step 3：用户自评
       ↓
Step 4：错题联动 → Step 5：批量/单章完成 → Step 6：复习报告
```

---

## 核心原则

| 原则 | 说明 |
|------|------|
| **不展示原文** | 直接输出精炼闪卡，不再展示深入底层知识 |
| **固定 6 段式结构** | 每张闪卡包含：核心概念 + 关键要点 + 常见误区 + 记忆口诀 + 关联知识 + 自测题 |
| **用户主导节奏** | 用户控制翻页速度，可随时暂停/跳过 |
| **错题优先** | 批量复习时，薄弱知识点优先展示 |
| **轻量自评** | 用户用 ✅/⚠️/❌ 标记置信度，不逐题打分 |

---

## Step 1：加载章节

### 1.1 单章模式

```markdown
## ⚡ 闪卡复习模式

正在加载章节：**[章节名称]**

本章节共 **X 个小节**，预计耗时 **Y 分钟**。

准备好后回复「开始」。
```

### 1.2 批量模式

```markdown
## ⚡ 批量闪卡复习

已加载你的学习进度：
- 已学章节：[章节 1]、[章节 2]、[章节 3]
- 错题知识点：[错题知识点 1]、[错题知识点 2]

**复习策略：**
1. 优先复习错题相关章节
2. 按学习顺序依次复习
3. 每章完成后自动继续

准备好后回复「开始」。
```

### 1.3 错题模式

```markdown
## ⚡ 错题闪卡复习

你的错题本共有 **X 道错题**，涉及以下知识点：

| 知识点 | 错误次数 | 最近错误日期 |
|--------|----------|--------------|
| [知识点 1] | N 次 | [日期] |
| [知识点 2] | N 次 | [日期] |

**策略：** 为每个薄弱知识点生成针对性闪卡，连续 2 次答对则标记已掌握。

准备好后回复「开始」。
```

---

## Step 2：生成闪卡

### 闪卡输出格式

```markdown
---
## ⚡ 闪卡 [N/M]：[小节名称]

### 📌 核心概念
[一句话定义，不超过 30 字]

### 🔑 关键要点
- [要点 1]
- [要点 2]
- [要点 3]
- [要点 4]（可选）
- [要点 5]（可选）

### ⚠️ 常见误区
- [误区 1：描述 + 正确理解]
- [误区 2：描述 + 正确理解]（可选）

### 💡 记忆口诀
[一句话帮助记忆，可以是口诀/比喻/联想]

### 🔗 关联知识
- 与 [知识 A] 的关系：[简述]
- 与 [知识 B] 的关系：[简述]

---

### ❓ 快速自测

[1 道判断题或选择题]

**选项：**
- A. [选项 A]
- B. [选项 B]

回复你的答案（A 或 B），或直接回复：
- 「✅ 掌握」→ 进入下一张闪卡
- 「⚠️ 模糊」→ 展开详细解释
- 「❌ 忘了」→ 展开详细解释 + 加入错题本
---
```

### 闪卡展开逻辑

当用户回复「⚠️ 模糊」或「❌ 忘了」时：

```markdown
### 📖 详细展开

**[小节名称]** 核心内容回顾：

[展示相关文档原文片段]

**正确理解：**
- [针对用户疑惑的详细解释]

**示例：**
[给出一个帮助理解的代码/生活示例]

---

回复「继续」进入下一张闪卡。
```

---

## Step 3：用户自评

### 置信度标记

| 标记 | 含义 | 后续动作 |
|------|------|----------|
| **✅ 掌握** | 完全理解，无需复习 | 标记已掌握，继续下一张 |
| **⚠️ 模糊** | 有部分理解但不确定 | 展开详细解释，标记待复习 |
| **❌ 忘了** | 完全不知道或记错 | 展开详细解释，加入错题本 |

---

## Step 4：错题联动

### 错题掌握判定

| 条件 | 动作 |
|------|------|
| 同一错题连续 **2 次** 答对 | 标记为「已掌握」，移出高频错题 |
| 同一错题连续 **3 次以上** 答错 | 标记为「顽固错题」，建议切换标准模式深度学习 |
| 新答错的题目 | 加入错题本，下次复习优先展示 |

---

## Step 5：章节/批量完成

### 单章完成

```markdown
## ✅ 本章闪卡复习完成

### 📊 复习结果

| 指标 | 数值 |
|------|------|
| 闪卡数量 | [N] 张 |
| ✅ 掌握 | [X] 张 ([X]%) |
| ⚠️ 模糊 | [X] 张 ([X]%) |
| ❌ 忘了 | [X] 张 ([X]%) |
| 自测正确率 | [X]% |

### 📋 待复习清单
- [ ] [知识点 1] - ⚠️ 模糊
- [ ] [知识点 2] - ❌ 忘了（已加入错题本）

### 接下来做什么？
- 回复「**继续**」→ 复习下一章节
- 回复「**展开 [知识点]**」→ 深度学习该知识点
- 回复「**结束**」→ 生成复习报告
```

### 批量完成

```markdown
## ✅ 批量闪卡复习完成

### 📊 总复习结果

| 指标 | 数值 |
|------|------|
| 复习章节数 | [N] 章 |
| 闪卡总数 | [N] 张 |
| 整体掌握度 | [XX]% |
| 新增错题 | [N] 道 |
| 已掌握错题 | [N] 道 |

### 🔥 顽固知识点
- [知识点 1]（连续 N 次复习标记为 ❌）
- [知识点 2]（连续 N 次复习标记为 ❌）

是否生成详细复习报告？
```

## 与现有模式的对比

| 维度 | 标准模式 | 快速模式 | 闪卡模式 |
|------|----------|----------|----------|
| **内容展示** | 展示原文 | 展示原文 | 不展示原文，直接给卡片 |
| **交互方式** | Agent 主导 | Agent 主导 | 用户主导 |
| **预计耗时** | 30-40 分钟/章 | 15-20 分钟/章 | 3-5 分钟/章 |
| **评分方式** | 0-5 分逐题评分 | 0-5 分逐题评分 | ✅/⚠️/❌ 置信度自评 |
| **题目数量** | 5 题/章 | 3 题/章 | 1 题/卡 |
| **适用场景** | 首次学习 | 复习/简单概念 | 考前冲刺/通勤复习 |

## 模式切换

| 切换方向 | 保留的上下文 | 清除/新增 |
|----------|-------------|----------|
| **→ 闪卡模式** | 已学章节、错题本、章节进度 | 新增 cardResults、batchMode |
| **闪卡 → 标准模式** | 已学章节、错题本、标记模糊的知识点 | 清除 cardResults，恢复完整评分体系 |
| **闪卡 → 快速模式** | 已学章节、错题本、标记模糊的知识点 | 清除 cardResults，恢复快速模式流程 |

**切换指令：**
- 「切换到标准模式」→ 进入标准模式 7 步流程
- 「切换到快速模式」→ 进入快速模式 5 步流程
- 「切换到闪卡模式」→ 进入闪卡模式流程
- 「闪卡复习」→ 对当前章节启动闪卡复习
- 「批量闪卡复习」→ 对已学章节批量闪卡复习
- 「复习错题」→ 进入错题闪卡复习
