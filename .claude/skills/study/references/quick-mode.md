# 快速模式流程模板（5 步精简）

**适用场景：** 复习或简单概念
**预计耗时：** 15-20 分钟

---

## 流程概览

```
Step 1：概念学习 → Step 2：尝试讲解 → Step 3：逐题测验
       ↓
Step 4：总结评分 → Step 5：复习计划
```

---

## Step 1：概念学习（智能展示 + 知识点范围）

**展示策略：** 详见 `references/smart-chapter-display.md`

**核心逻辑：**
```javascript
// 1. 加载章节后评估行数
const totalLines = chapterContent.split('\n').length;

// 2. 根据行数选择展示策略
if (totalLines < 150) {
  // 展示整章
  display = chapterContent;
} else if (totalLines < 400) {
  // 展示一个小节
  display = extractFirstSection(chapterContent);
} else {
  // 展示小节的前 65%
  display = truncateSection(extractFirstSection(chapterContent), 0.65);
}
```

## Step 3：逐题测验（核心环节 — 变式引擎集成）

### 测验前回顾

**核心要求：** 每次测验前必须重新展示相关原文，不准简略展示。

```markdown
## 📖 测验前回顾

在开始测验前，请快速回顾以下内容：

---

[重新展示当前学习的小节内容 - 与 Step 1 展示的完全相同]

---

准备好后，开始测验。
```

### 测验说明

- 每次只出 **1 道题**
- 根据回答评分（0-5 分）
- 根据评分决定追问或进入下一题
- **题目必须覆盖知识点清单中的所有知识点**
- **出题角度 = 记忆型/应用型/分析型（快速模式精简版，详见 `references/variation-engine.md`）**
- **快速模式默认使用基础难度（★☆☆）**

### 出题角度分布（快速模式）

| 题号 | 角度 | 目标 |
|------|------|------|
| 1 | 记忆型 | 检查概念记忆 |
| 2 | 应用型 | 检查实际运用 |
| 3 | 综合型（记忆+应用） | 检查整体掌握 |

### 出题范围约束 ⚠️

**重要：所有题目必须严格基于 Step 1 中展示的知识点清单。**

**出题前 AI 必须执行内部验证（集成变式引擎）：**

```javascript
// 伪代码逻辑 - AI 内部执行
const currentKnowledgePoints = state.currentChapterKnowledgePoints;
const mastery = state.knowledgePointMastery;
const questionTopic = 本题考察的知识点;
const questionAngle = 本题提问角度;  // 快速模式默认：memory/application

// 验证 1：知识点是否在清单内
if (!currentKnowledgePoints.some(kp => kp.name === questionTopic)) {
    generateNewQuestion(currentKnowledgePoints, mastery);
    return;
}

// 验证 2：标记已覆盖
if (mastery[questionTopic]) {
    mastery[questionTopic].angles[questionAngle].asked = true;
    mastery[questionTopic].angles[questionAngle].timesAsked++;
}
```

**如当前章节内容不足以出满 3 题：**
- 减少题数（如只出 2 题）
- 在总结中说明「本章内容较少，建议结合实际项目练习」
- 明确告知用户「本章共有 X 个知识点，已出 Y 题覆盖」

**用户纠正机制：**
- 如用户回复"超出范围"，AI 应立即：
  1. 道歉并承认错误
  2. 重新检查知识点清单
  3. 更换为清单内的题目
  4. 在后续出题时更加谨慎

### 出题格式

```markdown
## 逐题测验

**📊 知识点覆盖进度：** [已覆盖 X/总 Y 个知识点]

**问题 1/3** [题型：记忆型/应用型/综合型]  
**考察知识点：** [知识点名称]

[问题内容]

请回答：
```

### AI 评分反馈格式

```markdown
### 评分：[X]/5 [等级]

**✅ 正确部分：**
- [具体正确点]

**⚠️ 需要补充/纠正：**
- [遗漏或错误点]
- [正确理解应该是...]

**📌 下一步：**
[追问/进入下一题/详细讲解]
```

### 追问策略（快速模式）

| 评分 | 下一步 |
|------|--------|
| **5 分** | 进入下一题 |
| **4 分** | 简单确认后进入下一题 |
| **3 分** | 简要补充后进入下一题 |
| **2 分及以下** | 简要讲解后重新测试 |

### 题目类型分布（快速模式）

| 题号 | 题型 | 目标 |
|------|------|------|
| 1 | 基础 | 检查概念记忆 |
| 2 | 应用 | 检查实际运用 |
| 3 | 综合 | 检查整体掌握 |

### 错题本功能（集成变式引擎）

**AI 内部维护错题记录，追踪角度维度：**

```javascript
state.errorBook = {
  wrongQuestions: [
    {
      chapter: "章节名",
      knowledgePoint: "知识点名称",
      angle: "application",          // 做错的角度
      question: "题目内容",
      userAnswer: "用户回答",
      correctAnswer: "正确答案",
      score: 2,
      date: "2026-04-02"
    }
  ]
}
```

**复习模式中优先重做错题（变式引擎）：**
- 输入「复习错题」→ 从错题本中抽取题目，**换一个角度**重新问
- 同一知识点在不同角度都答对 → 掌握等级升级

## 章节完成检查点（增强版）

完成一个章节后（Step 4 总结评分后），执行以下检查：

1. **显示当前进度**

   - 剩余章节：[章节列表]

2. **询问用户**

3. **如选择「一键学完」**
   - 自动进入下一章节
   - 每章完成后自动继续
   - 所有章节完成后生成总报告

## Step 5 结束后：章节切换检查

**完成本章学习后，执行章节切换检查：**

```markdown
## 章节进度检查

✅ **本章学习完成！**

### 当前进度
- 已学章节：[本章名称]
- 剩余章节：[下一章名称]、[后续章节...]

### 接下来想做什么？
- 回复「**继续**」→ 学习下一章：[下一章名称]
- 回复「**跳过 [章节名]**」→ 跳过指定章节
- 回复「**结束**」→ 生成总学习报告，结束学习
- 回复「**复习**」→ 复习本章内容
```

**用户回复后的处理逻辑：**

| 用户回复 | 处理逻辑 |
|----------|----------|
| 「继续」 | 加载下一章内容，从 Step 1 重新开始 |
| 「跳过 [章节]」 | 标记该章节为跳过，进入下一章节 |
| 「结束」 | 生成总学习报告（见 study-summary-template.md） |
| 「复习」 | 重新进入 Step 1，复习本章内容 |

**进度追踪变量（内部维护）：**
```
- learnedChapters: []      // 已学章节
- skippedChapters: []      // 已跳过章节
- currentChapter: ""       // 当前章节
- remainingChapters: []    // 剩余章节
```
