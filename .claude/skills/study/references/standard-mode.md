# 标准模式流程模板（7 步深度学习）

**适用场景：** 深度学习新概念
**预计耗时：** 30-40 分钟

---

## 流程概览

```
Step 1：概念学习 → Step 2：尝试讲解 → Step 3：查漏补缺
       ↓
Step 4：简化重构 → Step 5：逐题测验 → Step 6：费曼复述
       ↓
Step 7：总结评分
```

---

## Step 1：概念学习（完整内容展示 + 知识点范围）

```markdown
## Step 1：明确学习目标

**章节：** [章节名称]
**小节：** [小节列表]

---

### 📖 完整内容展示

**⚠️ 必须展示完整章节内容，而非摘要！**

[读取并展示知识库中该章节的完整内容，必须包括：]
- ✅ 所有概念定义
- ✅ 所有工作原理说明
- ✅ 所有代码示例
- ✅ 所有注意事项/最佳实践
- ✅ 所有常见误区

---

### 🎯 知识点清单（出题范围边界）

根据上述内容，本章可出题的知识点包括：

| 编号 | 知识点 | 所属小节 | 题型建议 |
|------|--------|----------|----------|
| 1 | [知识点 1] | [小节名] | [基础/应用/分析] |
| 2 | [知识点 2] | [小节名] | [基础/应用/分析] |
| 3 | [知识点 3] | [小节名] | [基础/应用/分析] |

**⚠️ 重要：** 后续所有测验题目必须严格限制在上述知识点范围内！

---

### 📝 学前诊断

阅读完成后，请告诉我：
1. 你认为这个概念的核心是什么
2. 你之前对这个概念有什么了解或误解

```

## Step 5：逐题测验（核心环节）

### 测验说明

- 每次只出 **1 道题**
- 根据回答评分（0-5 分）
- 根据评分决定追问或进入下一题
- **题目必须覆盖知识点清单中的所有知识点**

### 出题范围约束 ⚠️

**重要：所有题目必须严格基于 Step 1 中展示的知识点清单。**

**出题前 AI 必须执行内部验证：**

```javascript
// 伪代码逻辑 - AI 内部执行
const currentKnowledgePoints = state.currentChapterKnowledgePoints;
const questionTopic = 本题考察的知识点;

// 验证 1：知识点是否在清单内
if (!currentKnowledgePoints.some(kp => kp.name === questionTopic)) {
    // 知识点超出范围，更换题目
    generateNewQuestion(currentKnowledgePoints);
    return;
}

// 验证 2：是否重复出题
if (state.questionHistory.includes(questionTopic)) {
    // 已出过该知识点，更换
    generateNewQuestion(currentKnowledgePoints);
    return;
}

// 验证 3：标记已覆盖知识点
currentKnowledgePoints.find(kp => kp.name === questionTopic).covered = true;
state.questionHistory.push(questionTopic);
```

**如当前章节内容不足以出满 5 题：**
- 减少题数（如只出 3 题）
- 在总结中说明"本章内容较少，建议结合实际项目练习"
- 明确告知用户「本章共有 X 个知识点，已出 Y 题覆盖」

**用户纠正机制：**
- 如用户回复"超出范围"，AI 应立即：
  1. 道歉并承认错误
  2. 重新检查知识点清单
  3. 更换为清单内的题目
  4. 在后续出题时更加谨慎

### 出题格式

```markdown
## 逐题测验

**📊 知识点覆盖进度：** [已覆盖 X/总 Y 个知识点]

**问题 1/5** [题型：基础/应用/分析]  
**考察知识点：** [知识点名称]

[问题内容]

请回答：
```

### AI 评分反馈格式

```markdown
### 评分：[X]/5 [等级]

**✅ 正确部分：**
- [具体正确点]

**⚠️ 需要补充/纠正：**
- [遗漏或错误点]
- [正确理解应该是...]

**📌 下一步：**
[追问/进入下一题/详细讲解]
```

### 追问策略

| 评分 | 下一步 |
|------|--------|
| **5 分** | 进入下一题 |
| **4 分** | 简单追问确认后进入下一题 |
| **3 分** | 补充讲解后，换角度再问 |
| **2 分及以下** | 详细讲解 + 示例，重新测试 |

## Step 7 结束后：章节切换检查

**完成本章学习后，执行章节切换检查：**

```markdown
## 章节进度检查

✅ **本章学习完成！**

### 当前进度
- 已学章节：[本章名称]
- 剩余章节：[下一章名称]、[后续章节...]

### 接下来想做什么？
- 回复「**继续**」→ 学习下一章：[下一章名称]
- 回复「**跳过 [章节名]**」→ 跳过指定章节
- 回复「**结束**」→ 生成总学习报告，结束学习
- 回复「**复习**」→ 复习本章内容
```

**用户回复后的处理逻辑：**

| 用户回复 | 处理逻辑 |
|----------|----------|
| 「继续」 | 加载下一章内容，从 Step 1 重新开始 |
| 「跳过 [章节]」 | 标记该章节为跳过，进入下一章节 |
| 「结束」 | 生成总学习报告（见 study-summary-template.md） |
| 「复习」 | 重新进入 Step 1，复习本章内容 |
