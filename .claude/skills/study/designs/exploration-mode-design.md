> **设计日期：** 2026-04-02

> **设计模式：** Pipeline + Inversion + Tool Wrapper

---

## 1. 设计概述

### 1.1 新模式定位

**探索模式（Exploration Mode）** 是 study skill 的第三种学习模式，专注于：
- **深度问答**：用户基于章节内容提问，Agent 解答
- **联网验证**：Agent 不确定时自动搜索官方来源
- **知识共建**：有价值的内容经用户确认后补充到文档
- **图表辅助**：涉及流程/架构/状态/交互时自动使用 Mermaid 绘制图表

### 1.2 与现有模式对比

| 模式 | 流程 | 交互方式 | 适用场景 | 预计耗时 |
|------|------|----------|----------|----------|
| **标准模式** | 7 步深度学习 | Agent 主导流程 | 首次学习复杂概念 | 30-40 分钟 |
| **快速模式** | 5 步精简 | Agent 主导流程 | 复习/简单概念 | 15-20 分钟 |
| **探索模式** | 问答驱动 + 图表 | 用户主导提问 | 深度研究/文档共建 | 灵活 |

### 1.3 设计模式选择

**组合模式：Pipeline + Inversion + Tool Wrapper**

| 模式 | 应用点 |
|------|--------|
| **Pipeline** | 固定流程：展示→提问→解答→搜索→补充 |
| **Inversion** | Agent 主动询问用户是否确认补充文档 |
| **Tool Wrapper** | 按需加载 web-access 技能进行联网搜索，按需使用 Mermaid 绘制图表 |

### 1.4 Mermaid 图表使用规范（与 research skill 保持一致）

**参考规范：** `../../research/references/mermaid-guide.md`

**图表类型选择决策树：****

```
需要画什么类型的图？
│
├── 流程/步骤/顺序 → flowchart LR（从左到右）或 flowchart TD（从上到下）
│
├── 系统架构/分层 → flowchart TB + subgraph 分组
│
├── 状态变化/状态机 → stateDiagram-v2
│
├── 组件交互/时序 → sequenceDiagram
│
├── 数据结构/链表树 → flowchart LR + subgraph
│
├── 概念分层/思维导图 → mindmap
│
└── 项目计划/时间线 → gantt
```

**图表使用场景：**

| 场景类型 | 图表类型 | 示例 |
|----------|----------|------|
| **流程步骤** | flowchart LR/TD | 置信度评估流程、搜索决策流程 |
| **系统架构** | flowchart TB + subgraph | 分层架构、模块关系 |
| **状态变化** | stateDiagram-v2 | 生命周期、状态机 |
| **组件交互** | sequenceDiagram | API 调用、用户操作响应 |
| **数据结构** | flowchart LR | 链表、树、Fiber 结构 |

**图表绘制规范（与 research skill 保持一致）：**
- ✅ 必须使用 Mermaid，禁止 ASCII 字符画图
- ✅ 参考规范：`../../research/references/mermaid-guide.md`
- ✅ 节点数控制在 5-15 个，超过则拆分或使用 subgraph 分组
- ✅ 使用有意义的节点命名，避免 A/B/C 等无意义标签
- ✅ 连接线标注简洁，复杂说明使用 note 注释
- ✅ 图表自动布局，永不错位
- ✅ 图表类型选择遵循决策树（流程→flowchart、状态→stateDiagram-v2、时序→sequenceDiagram）

#### 阶段 2：用户提问与解答

```markdown
## ❓ 问题：[用户问题]

### Agent 思考过程（内部）

1. **判断问题类型**
   - 概念定义 → 直接回答
   - 原理深入 → 直接回答 + 示例
   - 事实验证 → 检查置信度
   - 扩展知识 → 可能需要搜索

2. **置信度评估**
   - 高置信度（文档有明确答案）→ 直接回答
   - 中置信度（文档有部分信息）→ 回答 + 标注不确定
   - 低置信度（文档无信息或需要验证）→ 触发搜索

3. **搜索决策**
   IF (置信度 == 低) OR (用户要求验证) OR (需要最新信息) THEN
     调用 web-access 搜索
     搜索优先级：官方文档 > GitHub > 权威博客 > 社区
   END IF
```

**搜索触发条件：**

| 条件 | 说明 | 示例 |
|------|------|------|
| **文档无明确答案** | 知识点在文档中未覆盖 | 新版本特性 |
| **需要验证准确性** | Agent 不确定答案是否正确 | API 参数变更 |
| **需要最新信息** | 文档可能过时 | 框架最新版本 |
| **用户明确要求** | 用户要求查官方确认 | 「请查官方文档」 |

**搜索执行逻辑：**

```javascript
async function searchIfNeeded(question, confidence) {
  if (confidence < THRESHOLD) {
    // 调用 web-access 技能
    const searchResult = await invokeWebAccess({
      query: question + " site:react.dev OR site:github.com",
      priority: ["official-doc", "github", "authority-blog"]
    });
    
    // 提取来源链接
    return {
      answer: searchResult.summary,
      sources: searchResult.urls,
      confidence: "verified"
    };
  }
  return null;
}
```

**写入文档逻辑：**

```javascript
function writeToDocument(content, location) {
  // 1. 读取目标文档
  const doc = readDocument(location.file);
  
  // 2. 找到目标章节
  const section = findSection(doc, location.section);
  
  // 3. 插入内容
  section.content += `\n\n---\n\n## ${content.title}\n\n${content.body}\n\n> 来源：${content.source} | 补充日期：${content.date}`;
  
  // 4. 更新文档更新时间
  doc.lastUpdated = new Date().toISOString();
  
  // 5. 写入文件
  writeDocument(location.file, doc);
  
  return { success: true, path: location.file };
}
```

## 4. 与现有模式的关系

### 4.1 模式对比

| 维度 | 标准模式 | 快速模式 | 探索模式 |
|------|----------|----------|----------|
| **主导方** | Agent 主导 | Agent 主导 | 用户主导 |
| **流程固定性** | 严格 7 步 | 严格 5 步 | 灵活问答 |
| **出题机制** | 必须出题 | 必须出题 | 不出题 |
| **评分机制** | 有评分 | 有评分 | 无评分 |
| **联网搜索** | 不需要 | 不需要 | 需要 |
| **文档补充** | 不补充 | 不补充 | 可补充 |
| **适用场景** | 深度学习 | 复习 | 研究/共建 |

### 4.2 模式切换

```markdown
**在学习过程中可切换模式：**

- 「切换到标准模式」→ 进入标准模式流程
- 「切换到快速模式」→ 进入快速模式流程
- 「切换到探索模式」→ 进入探索模式流程

**切换时保留上下文：**
```

## 13. 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| **过度搜索** | 响应慢、token 消耗大 | 设置置信度阈值，高置信度不搜索 |
| **文档污染** | 低质量内容写入 | 必须用户确认后才写入 |
| **来源可靠性** | 搜索到不可靠来源 | 严格优先级：官方 > GitHub > 博客 |
| **上下文溢出** | 长对话超出限制 | Sidecar 持久化，定期总结 |
