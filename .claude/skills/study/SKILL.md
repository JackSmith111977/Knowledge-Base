# study - 费曼学习法互动教学 Skill

## 用途

基于**费曼学习法**的互动教学 Skill，提供四种学习模式：

| 模式 | 流程 | 交互方式 | 适用场景 | 预计耗时 |
|------|------|----------|----------|----------|
| **标准模式** | 7 步：概念→讲解→查漏→重构→逐题测验→复述→评分 | Agent 主导 | 深度学习新概念 | 30-40 分钟 |
| **快速模式** | 5 步：概念→讲解→逐题测验→评分 | Agent 主导 | 复习/快速浏览 | 15-20 分钟 |
| **闪卡模式** | 6 段式卡片：核心概念→要点→误区→口诀→关联→自测 | 用户主导 | 考前冲刺/通勤复习 | 3-5 分钟/章 |
| **探索模式** | 问答驱动：展示→提问→解答→搜索→补充 | 用户主导 | 深度研究/文档共建 | 灵活 |

**核心原则：** 以教促学、逐题追问、评分反馈、小节覆盖

**新增特性（v5.2.0）：**
- ⚡ 闪卡模式：6 段式卡片速记（核心概念/要点/误区/口诀/关联/自测），3-5 分钟/章
- 🔁 批量复习：连续复习已学章节，错题优先展示
- 🎯 置信度自评：✅/⚠️/❌ 轻量标记，不逐题打分
- 🔗 错题联动：自测答错自动加入错题本，连续答对标记已掌握
- 🔍 探索模式：问答驱动，支持联网搜索和文档共建
- 📡 联网验证：Agent 不确定时自动搜索官方来源
- 📝 知识共建：有价值的内容经确认后补充到文档

## 工作流程

### 阶段 0：启动与模式选择

**步骤 0.1：欢迎与模式选择**

```markdown
📚 **费曼学习法学习向导**

请选择学习模式：

**🐢 标准模式**（约 30-40 分钟）
- 7 步流程：概念→讲解→查漏→重构→逐题测验→复述→评分
- 包含比喻、深度复述、详细评分
- 适合：首次学习复杂概念

**🐇 快速模式**（约 15-20 分钟）
- 5 步流程：概念→讲解→逐题测验→评分
- 精简流程，聚焦核心测验
- 适合：复习或简单概念

**⚡ 闪卡模式**（约 3-5 分钟/章）
- 6 段式卡片：核心概念→关键要点→常见误区→记忆口诀→关联知识→自测
- 不展示原文，直接精炼卡片
- 支持批量复习、错题联动、置信度自评
- 适合：考前冲刺、通勤复习、知识唤醒

**🔍 探索模式**（灵活）
- 问答驱动：展示章节→自由提问→Agent 解答→文档共建
- 支持联网搜索官方来源
- 适合：深度研究、文档补充

请回复「标准」、「快速」、「闪卡」或「探索」选择模式。
```

**步骤 0.2：主题确认**

- **方式 A**：用户指定主题
- **方式 B**：扫描知识库推荐

**步骤 0.3：知识库内容加载**

加载成功后，显示章节结构，用户确认开始。

### 阶段 1-3：学习流程（分模式）

**标准模式：** 详见 `references/standard-mode.md`
**快速模式：** 详见 `references/quick-mode.md`
**闪卡模式：** 详见 `references/flashcard-mode.md`
**探索模式：** 详见 `references/exploration-mode.md`

**标准/快速模式核心机制：**
| 机制 | 说明 |
|------|------|
| **逐题追问** | 每次只问 1 题，根据回答决定下一题 |
| **评分反馈** | 每题 0-5 分，附点评和纠正 |
| **小节覆盖** | 按知识库小节结构逐个击破 |
| **章节追踪** | 记录已学章节，完成后询问是否继续 |
| **知识点清单** | Step 1 生成可出题知识点，防止超纲 |
| **出题验证** | 每次出题前验证知识点是否在清单内 |
| **错题本** | 记录评分≤3 分的题目，复习时优先重做 |

**闪卡模式核心机制：**
| 机制 | 说明 |
|------|------|
| **6 段式卡片** | 核心概念 + 关键要点 + 常见误区 + 记忆口诀 + 关联知识 + 自测题 |
| **置信度自评** | ✅/⚠️/❌ 轻量标记，不逐题打分 |
| **错题联动** | 自测答错自动加入错题本，连续 2 次答对标记已掌握 |
| **批量复习** | 连续复习已学章节，错题相关章节优先展示 |
| **不展示原文** | 直接输出精炼卡片，仅在用户标记模糊/忘了时展开 |

**探索模式核心机制：**
| 机制 | 说明 |
|------|------|
| **置信度评估** | 每题评估置信度（高≥0.8, 中 0.5-0.7, 低<0.5），决定是否需要搜索 |
| **联网搜索** | 低置信度/用户要求/版本信息时触发 web-access 搜索官方来源 |
| **来源标注** | 所有解答标注来源（官方文档/当前文档/GitHub/社区） |
| **文档共建** | 有价值内容经用户确认后写入文档 |
| **会话记录** | 写入 sidecar 持久化（问题列表、搜索历史、补充记录） |

## 评分标准

详见 `references/scoring-rubric.md`

| 分数 | 标准 |
|------|------|
| **5 分** | 准确完整，能用自己的话解释，无知识盲点 |
| **4 分** | 基本正确，有小遗漏或表述不够清晰 |
| **3 分** | 核心概念正确，有明显遗漏或误解 |
| **2 分** | 部分正确，但关键概念有错误 |
| **1 分** | 回答与问题关联度低 |
| **0 分** | 未回答或完全错误 |

## 踩坑清单

| 陷阱 | 错误做法 | 正确做法 |
|------|----------|----------|
| **术语堆砌** | 允许用户使用行话 | 强制用简单语言解释 |
| **跳过盲点** | 用户说不懂就直接给答案 | 引导用户自己发现并解决 |
| **一次性多问** | 一次抛出多个问题 | 逐题追问，根据回答调整 |
| **不评分** | 只给点评 | 每题必须打分，记录追踪 |
| **不复盘** | 学完就结束 | 必须生成总结和复习计划 |
| **章节切换缺失** | 完成一章直接结束 | 每章完成后询问是否继续 |
| **进度不追踪** | 不记录已学章节 | 维护 learnedChapters 列表 |
| **超纲出题** | 出后续章节的内容 | 题目严格限制在 Step 1 知识点清单内 |
| **展示不完整** | Step 1 只展示摘要 | Step 1 必须展示完整章节内容 |
| **错题不记录** | 不做错题记录 | 维护 errorBook，复习时优先重做 |
| **简略展示** | 展示 AI 概括的内容 | 必须展示文档原文，不准简略 |
| **不看长度** | 总是展示整章 | 根据内容长度选择展示整章/小节/小节部分 |
