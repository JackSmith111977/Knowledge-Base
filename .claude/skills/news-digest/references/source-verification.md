# 来源可信度验证指南

> Reviewer 模式：验证新闻来源是否可信，追溯原始出处

---

## 验证流程

### 第 1 步：白名单匹配

检查来源是否在 `config.json` 中已配置：

```
IF source.name IN config.topics[].sources[]
  → 使用配置的 weight 作为基础分
  → 进入第 1.5 步
ELSE
  → 进入第 2 步（未知来源评估）
```

### 第 1.5 步：原始出处追溯（新增 v1.2.0）

**这是防止「事实来源不标注」问题的核心步骤。**

```
IF 当前来源是转载/聚合站（百家号、知乎专栏、CSDN 转载、腾讯网、搜狐网、网易号等）
  → 必须尝试从正文中识别原始出处：
    - 文章开头/结尾的「本文来自 XX」「原载于 XX」标注
    - 作者署名（如非站点自有作者）
    - 原文链接指向的外部地址
    - 文章中引用的数据来源声明
  → 如能识别原始出处：
    - 记录 originalSource 字段
    - 使用原始出处的 weight 作为来源权威性评分依据
    - 当前转载站的 weight 作为次级参考
  → 如无法识别原始出处：
    - 标记为「原始出处未知」
    - 来源权威性评分上限为 6 分
    - 在报告中标注「转载，原始出处未追溯」
ELSE
  → 当前来源即为原始出处，正常评分
```

**常见转载/聚合站列表（需要追溯）：**
- 百家号（baijiahao.baidu.com）— 几乎全部为转载
- 腾讯新闻（news.qq.com）— 转载聚合
- 搜狐新闻（news.sohu.com）— 转载聚合
- 网易新闻（163.com/dy/）— 自媒体转载
- 知乎专栏（zhuanlan.zhihu.com）— 部分为原创，需判断
- CSDN 博客（blog.csdn.net）— 社区转载常见
- 新浪博客/财经（finance.sina.com.cn）— 部分为转载

**无需追溯的原创/首发站：**
- 机器之心、量子位、36kr、虎嗅（原创科技媒体）
- TechCrunch、The Verge、Ars Technica（原创英文科技媒体）
- Anthropic Blog、OpenAI Blog、Google AI Blog（官方博客）
- HackerNews（社区原创）
- 官方公告、论文发布源

### 第 2 步：未知来源评估

对未在配置中的来源，按以下标准评估：

#### 域名检查

| 检查项 | 通过标准 | 失败处理 |
|--------|----------|----------|
| 域名年龄 | 注册 > 1 年 | 标记为可疑，-2 分 |
| HTTPS | 有效 SSL 证书 | 标记为不安全，-3 分 |
| 备案信息 | 有 ICP 备案（中文站） | 无法确认身份，-1 分 |

#### 内容特征

| 检查项 | 可信特征 | 不可信特征 |
|--------|----------|------------|
| 作者署名 | 有明确作者和编辑 | 无署名或 AI 生成标记 |
| 引用来源 | 有引用链接和数据 | 纯观点无依据 |
| 纠错机制 | 有更正/勘误记录 | 从不更正 |
| 广告标识 | 广告/赞助内容明确标注 | 软文混在新闻中 |

### 第 3 步：URL 结构验证

```
✅ 正常新闻 URL 特征：
- /news/, /article/, /p/, /post/ 等路径
- 包含日期或 ID
- 无异常参数（如 ?ref=spam）

❌ 异常 URL 特征：
- 短链接（t.cn, bit.ly 等，需展开后验证）
- 包含大量追踪参数（utm_source 等超过 3 个）
- 域名与已知新闻站点不匹配
- URL 路径包含 login/paywall 等关键词
```

### 第 4 步：交叉验证

```
IF 同一新闻出现在 >= 2 个可信来源
  → 提高可信度 +0.5 分
  → 记录多信源报道标记
ELSE IF 仅单一来源报道
  → 标记为「独家」，需要更高的内容深度分
ELSE IF 其他来源明确辟谣
  → 标记为「存疑」，评分上限 3 分（将被过滤）
```

## 可信来源分类参考

### Tier 1 — 权威媒体（直接信任）
- 新华社、人民日报、财新
- Reuters, AP, Bloomberg
- TechCrunch, The Verge, Ars Technica

### Tier 2 — 专业科技媒体（高信任）
- 36kr、机器之心、量子位、InfoQ
- HackerNews, Dev.to
- GitHub Blog, Google AI Blog
- 凤凰网、虎嗅

### Tier 3 — 社区/博客（需内容评估）
- 掘金、CSDN、博客园
- Medium 个人博客
- Reddit 技术板块

### Tier 4 — 社交媒体（低信任，需交叉验证）
- Twitter/X
- 微博、微信公众号
- 论坛转载

### 转载/聚合站（需追溯原始出处）
- 百家号、腾讯新闻、搜狐新闻、网易号
- 知乎专栏（部分原创，需判断）
- CSDN 转载

### 黑名单（直接过滤）
- 已知假新闻站点
- 纯内容农场（无编辑团队）
- 被多次举报的自媒体
