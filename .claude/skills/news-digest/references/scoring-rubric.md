# 评分系统详细规则

> 新闻智能评分系统，5 维度加权计算，满分 10 分

---

## 评分公式

```
综合得分 = sourceAuthority × 0.25 + timeliness × 0.15 + relevance × 0.20 + contentDepth × 0.20 + factTraceability × 0.20
```

**门控阈值：** 综合得分 > 4.0 才纳入报告
**内容质量门控：摘要（基于全文获取后）< 50 字且无具体数据的条目直接过滤**

---

## 维度 1：来源权威性（Source Authority）— 权重 25%

### 评分标准

| 分数 | 条件 |
|------|------|
| 9-10 | config.json 中 source.weight >= 0.9 的白名单来源，**且原始出处可追溯** |
| 7-8 | 知名科技媒体（The Verge, Ars Technica, Wired, 36kr, 机器之心等），原创或首发 |
| 5-6 | 一般科技博客、技术社区，**或转载站但追溯到了可信原始出处** |
| 3-4 | 转载站且**无法追溯原始出处**，或社交媒体/论坛转载 |
| 1-2 | 来源不明、自媒体、内容农场 |

### 可信来源白名单

**高可信（weight 0.9-1.0）：**
- 36kr、机器之心、量子位（中文 AI）
- TechCrunch、The Verge、Ars Technica、Wired（英文科技）
- HackerNews（技术社区）
- 官方 Blog（Anthropic、OpenAI、Google AI、DeepMind）

**中可信（weight 0.7-0.89）：**
- 掘金、InfoQ、CSDN（中文技术社区）
- Dev.to、Medium 技术文章
- GitHub Trending
- 凤凰网科技、虎嗅

### 转载站追溯规则

- 转载站（百家号、腾讯新闻、搜狐、网易号等）且追溯到原始出处 → 使用原始出处 weight 评分
- 转载站且**无法追溯**原始出处 → 上限 6 分
- 来源在 config.json 中标记为 `suspicious: true` → 降 2 分

---

## 维度 2：时效性（Timeliness）— 权重 15%

### 评分标准

| 分数 | 条件 |
|------|------|
| 10 | 发布时间 <= 2 小时 |
| 8-9 | 发布时间 <= 6 小时 |
| 6-7 | 发布时间 <= 24 小时 |
| 4-5 | 发布时间 <= 48 小时 |
| 2-3 | 发布时间 <= 72 小时 |
| 1 | 发布时间 > 72 小时 |

### 注意

- 无法确定发布时间时，默认给 5 分
- 每日定时任务中，24h 前的新闻占大多数，可适当放宽

---

## 维度 3：相关性（Relevance）— 权重 20%

### 评分标准

| 分数 | 条件 |
|------|------|
| 9-10 | 标题和简介均包含主题关键词，且为核心讨论 |
| 7-8 | 标题或简介包含关键词，主要内容相关 |
| 5-6 | 间接相关（提到关键词但不是核心主题） |
| 3-4 | 弱相关（关键词仅作为背景提及） |
| 1-2 | 几乎不相关（关键词匹配但内容无关） |

### 关键词匹配算法

```
1. 标题完全匹配关键词 → +3 分
2. 简介包含关键词 → +2 分
3. 主题是文章核心（非仅提及）→ +2 分
4. 属于该主题的子领域 → +1 分
5. 与近期热点相关 → +1 分
```

---

## 维度 4：内容深度（Content Depth）— 权重 20%

### 评分标准

| 分数 | 条件 |
|------|------|
| 9-10 | 深度分析/评测/教程，有数据支撑，观点明确，**全文摘要 >= 200 字** |
| 7-8 | 详细报道，包含多方观点和具体信息，**全文摘要 >= 150 字** |
| 5-6 | 常规新闻报道，信息完整但无深度，**全文摘要 >= 100 字** |
| 3-4 | 简讯/快讯，信息量较少，**全文摘要 < 100 字但有具体数据** |
| 1-2 | 标题党、纯转载、信息碎片化，**全文摘要 < 50 字** |

### 内容深度判断指标（基于步骤 1.5 全文获取后的内容）

- 摘要字数 > 150 字 → 倾向于 7 分以上
- 包含具体数据/数字/引用 → +1 分
- 有明确的观点或分析 → +1 分
- 纯事件陈述无分析 → -1 分
- 标题党特征（夸张词汇、感叹号过多）→ -2 分

---

## 维度 5：事实标注完整性（Fact Traceability）— 权重 20%（新增 v1.2.0）

### 评分标准

| 分数 | 条件 |
|------|------|
| 9-10 | 标注了原始来源 + 作者 + 关键数据出处，转载内容标注了「原载于 XX」 |
| 7-8 | 标注了原始来源和作者，数据来源清晰可查 |
| 5-6 | 标注了来源站名，但未追溯原始出处（转载站） |
| 3-4 | 仅标注了转载站名，无原始出处信息 |
| 1-2 | 无来源标注或来源不明 |

### 评分细则

```
1. 标注了原始来源（非转载站名）→ +3 分
2. 标注了作者姓名/笔名 → +2 分
3. 转载内容标注了「原载于 XX」→ +2 分
4. 关键数据/统计有明确出处 → +1 分
5. 多信源报道标注了各信源信息 → +1 分
6. 无法追溯原始出处且未标注 → -3 分
```

### 强制要求

- **所有转载内容必须在报告中展示原始来源信息**
- 无法追溯原始出处的转载内容，「事实标注完整性」维度上限为 5 分
- 报告中每条新闻必须包含「来源」字段，转载内容额外包含「原始来源」字段

---

## 特殊规则

### 加分项

- 独家报道/首发 → +0.5 分
- 包含原始数据或研究 → +0.5 分
- 多语言来源（增加视角多样性）→ +0.3 分

### 扣分项

- 付费墙/登录墙 → -1 分
- 软文/广告特征 → -2 分
- 与已收录新闻重复度 > 80% → 标记重复，保留得分高的版本

---

## 评分示例

```
新闻："OpenAI 发布 GPT-6，性能提升 40%"
来源：凤凰网科技（转载，原载于机器之心）
作者：张三
发布时间：3 小时前
主题：AI 前沿
全文摘要：180 字，包含具体数据和多方观点

评分计算：
- sourceAuthority: 8 × 0.25 = 2.00（原始出处机器之心为高可信来源）
- timeliness: 9 × 0.15 = 1.35（3 小时内）
- relevance: 9 × 0.20 = 1.80（标题简介均含关键词）
- contentDepth: 8 × 0.20 = 1.60（详细报道，180字摘要）
- factTraceability: 9 × 0.20 = 1.80（标注了原载于机器之心+作者）
综合得分 = 2.00 + 1.35 + 1.80 + 1.60 + 1.80 = 8.55

结果：通过（> 4.0）
```
