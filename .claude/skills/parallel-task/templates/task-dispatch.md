# 任务分发模板

## 模板使用说明

本模板用于定义如何分发任务给多个 SubAgent 并行执行。

---

## 模板 1：批量并行任务

```markdown
## 任务描述

并发执行以下独立子任务：

### 子任务清单
1. **任务 A:** [具体描述]
   - 目标：[预期结果]
   - 范围：[文件/目录]
   - 类型：[审查/分析/生成]
   - **输出位置：** [指定路径，如：`.work/[主题]/drafts/task-a.md`]

2. **任务 B:** [具体描述]
   - 目标：[预期结果]
   - 范围：[文件/目录]
   - 类型：[审查/分析/生成]
   - **输出位置：** [指定路径]

3. **任务 C:** [具体描述]
   - 目标：[预期结果]
   - 范围：[文件/目录]
   - 类型：[审查/分析/生成]
   - **输出位置：** [指定路径]

## 执行要求

- **执行模式：** 批量并行
- **预期输出：** 分 Agent 报告 + 整体汇总
- **时间要求：** [可选]
- **文件约束：** 所有过程文件必须写入指定的输出位置，禁止随意存放
```

---

## 模板 2：任务队列（有依赖关系）

```markdown
## 任务描述

按顺序执行以下存在依赖关系的任务：

### 任务依赖图

```
任务 A (前置)
  │
  ├──→ 任务 B (依赖 A)
  │      │
  │      └──→ 任务 C (依赖 B)
  │
  └──→ 任务 D (依赖 A)
```

### 任务清单

1. **任务 A:** [描述]
   - 类型：前置任务
   - 输出：[产物]
   - **输出位置：** [指定路径]

2. **任务 B:** [描述]
   - 依赖：任务 A 完成
   - 输入：任务 A 的输出
   - **输出位置：** [指定路径]

3. **任务 C:** [描述]
   - 依赖：任务 B 完成
   - 输入：任务 B 的输出
   - **输出位置：** [指定路径]

**文件约束：** 所有过程文件必须写入指定的输出位置，禁止随意存放
```

---

## 模板 3：混合模式

```markdown
## 任务描述

部分任务可并行，部分需要顺序执行：

### 并行组 A

可同时执行：
- 任务 A1: [描述]
  - **输出位置：** [指定路径]
- 任务 A2: [描述]
  - **输出位置：** [指定路径]
- 任务 A3: [描述]
  - **输出位置：** [指定路径]

### 顺序组 B（依赖并行组完成）

按顺序执行：
1. 任务 B1: [描述]（依赖 A1/A2/A3 完成）
   - **输出位置：** [指定路径]
2. 任务 B2: [描述]（依赖 B1 完成）
   - **输出位置：** [指定路径]

**文件约束：** 所有过程文件必须写入指定的输出位置，禁止随意存放
```

## 输出报告模板

```markdown
# [任务名称] 执行报告

## 执行概览

| 指标 | 数值 |
|------|------|
| 执行模式 | [批量并行/任务队列/混合] |
| SubAgent 数量 | N |
| 总耗时 | X 分钟 |
| 完成时间 | YYYY-MM-DD HH:mm |

---

## SubAgent 报告

### 报告 1: [SubAgent 名称]

**执行时间：** X 分钟
**任务描述：** [具体任务]
**执行范围：** [文件列表/目录]

#### 发现汇总
- 发现 1
- 发现 2

#### 详细结果
[具体内容]

#### 建议与后续
- 建议 1
- 建议 2

---

### 报告 2: [SubAgent 名称]
...

---

## 整体汇总

### 关键指标
| 指标 | 数值 |
|------|------|
| 总任务数 | N |
| 完成数 | M |
| 发现问题数 | K |

### 优先级行动项
1. **Critical:** [需要立即处理]
2. **Warning:** [需要注意]
3. **Info:** [改进建议]
```
