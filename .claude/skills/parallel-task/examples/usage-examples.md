# 使用示例

## 示例 1：批量代码审查

### 用户输入

```
/parallel-task 并发执行以下审查任务：
1. 审查 src/components 目录的代码规范
2. 检查 src/utils 目录的安全漏洞
3. 分析 src/api 目录的性能问题
```

### Skill 执行流程

```
1. 任务分析
   ├── 任务 1: 代码规范审查 → 委派给 code-reviewer
   ├── 任务 2: 安全漏洞检查 → 委派给 security-scanner
   └── 任务 3: 性能问题分析 → 委派给 performance-analyzer

2. 模式选择：批量并行（三个任务独立，无依赖）

3. 并发执行：同时启动 3 个 SubAgent

4. 结果收集：等待所有 SubAgent 完成

5. 报告生成：输出分 Agent 报告 + 整体汇总
```

### 预期输出

```markdown
# 批量代码审查执行报告

## 执行概览

**执行模式：** 批量并行
**SubAgent 数量：** 3
**总耗时：** 4.5 分钟

---

## SubAgent 报告

### 报告 1: 代码规范审查

**执行范围：** src/components/
**发现问题：** 12 个（Critical: 2, Warning: 5, Info: 5）

#### 关键问题
1. [Critical] src/components/Auth.jsx:45 - 未处理的空指针
2. [Critical] src/components/Form.jsx:89 - 资源泄漏风险

...

### 报告 2: 安全漏洞扫描
...

### 报告 3: 性能问题分析
...

---

## 整体汇总

### 优先级行动项
1. **Critical:** 修复 Auth.jsx 空指针问题
2. **Warning:** 优化 Form.jsx 资源管理
3. **Info:** 统一命名规范
```

## 示例 2：全面项目分析

### 用户输入

```
全面分析这个项目，检查所有潜在问题
```

### Skill 执行流程

```
1. 自动拆解任务
   ├── 安全漏洞扫描（SQL 注入、XSS、CSRF）
   ├── 性能问题检测（N+1 查询、内存泄漏）
   ├── 代码规范检查（命名、格式、注释）
   ├── 测试覆盖率分析
   └── 依赖安全检查

2. 模式选择：批量并行（5 个独立分析任务）

3. 并发执行：启动 5 个专项 SubAgent

4. 结果整合：5 份专项报告 + 综合建议
```

### 预期输出

```markdown
# 全面项目分析报告

## 执行概览

**执行模式：** 批量并行
**SubAgent 数量：** 5
**总耗时：** 8.2 分钟

---

## 专项报告

### 报告 1: 安全漏洞扫描
- SQL 注入风险：2 处
- XSS 漏洞：1 处
- CSRF 保护：已配置

### 报告 2: 性能问题检测
- N+1 查询：3 处
- 潜在内存泄漏：1 处

### 报告 3: 代码规范检查
- 命名不规范：15 处
- 缺少注释：8 处

...

---

## 综合建议

### 立即处理
1. 修复 SQL 注入漏洞
2. 优化 N+1 查询

### 后续改进
1. 统一命名规范
2. 补充关键注释
```

## 示例 3：大型重构任务（任务队列模式）

### 用户输入

```
将项目从 JavaScript 迁移到 TypeScript
```

### Skill 执行流程

```
1. 任务分析
   - 识别依赖关系
   - 无法完全并行

2. 模式选择：任务队列模式

3. 执行顺序
   (1) 分析项目结构 → 前置任务
   (2) 生成类型定义 → 依赖 (1)
   (3) 迁移前端文件 → 依赖 (2)
   (4) 迁移后端文件 → 依赖 (2)
   (5) 类型验证 → 依赖 (3)(4)

4. 进度追踪：实时更新每个阶段状态
```

### 预期输出

```markdown
# JS→TS 迁移执行报告

## 执行概览

**执行模式：** 任务队列
**总阶段数：** 5
**总耗时：** 25 分钟

---

## 阶段报告

### 阶段 1: 项目结构分析 ✅
**耗时：** 2 分钟
**输出：** 文件清单、依赖关系图

### 阶段 2: 类型定义生成 ✅
**耗时：** 5 分钟
**输出：** types/目录、全局类型声明

### 阶段 3: 前端文件迁移 ✅

**迁移文件：** 45 个 .jsx → .tsx

### 阶段 4: 后端文件迁移 ✅

### 阶段 5: 类型验证 ✅

---

## 迁移汇总

**剩余手动修复：** 2 处
**编译状态：** 通过
```

## 示例 4：混合模式（部分并行 + 部分顺序）

### 用户输入

```
并行分析代码问题，然后生成修复方案并验证
```

### Skill 执行流程

```
阶段 1（并行组）：
├── 安全分析
├── 性能分析
└── 规范检查

阶段 2（顺序组，依赖阶段 1）：
1. 整合问题清单 → 依赖阶段 1 完成
2. 生成修复方案 → 依赖问题清单
3. 执行修复验证 → 依赖修复完成
```

### 预期输出

```markdown
# 混合模式执行报告

## 执行概览

**执行模式：** 混合模式
**并行组：** 3 个任务
**顺序组：** 3 个任务
**总耗时：** 12 分钟

---

## 阶段 1: 并行分析

[3 份专项报告]

---

## 阶段 2: 修复与验证

### 步骤 1: 问题整合
汇总 15 个问题（Critical: 3, Warning: 7, Info: 5）

### 步骤 2: 修复方案生成
生成 15 个修复补丁

### 步骤 3: 修复验证
- 自动修复：12 处
- 手动待修复：3 处
- 验证通过：100%
```
