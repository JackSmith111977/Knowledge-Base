# 执行检查清单

## 执行前检查

### 任务分析

- [ ] 确认任务总数和具体描述
- [ ] 识别每个任务的类型（审查/分析/生成/其他）
- [ ] 确定每个任务的执行范围（文件/目录）
- [ ] 判断任务之间是否存在依赖关系

### 模式选择

- [ ] 所有任务独立 → 选择**批量并行**模式
- [ ] 任务存在依赖 → 选择**任务队列**模式
- [ ] 部分独立部分依赖 → 选择**混合**模式

### SubAgent 匹配

- [ ] 为每个任务选择合适的 SubAgent 类型
- [ ] 确认 SubAgent 配置存在（或可自动创建）
- [ ] 检查 SubAgent 工具权限是否充足

---

## 执行中检查

### 并发控制

- [ ] 并行任务数不超过推荐值（~10 个）
- [ ] 避免多个 SubAgent 同时编辑同一文件
- [ ] 只读任务可安全并行

### 进度追踪

- [ ] 实时更新每个 SubAgent 的执行状态
- [ ] 记录每个任务的开始时间和预计完成时间
- [ ] 标记延迟或异常的任务

### 异常处理

- [ ] 检测 SubAgent 执行失败
- [ ] 记录错误原因和上下文
- [ ] 决定是否重试或跳过

---

## 执行后检查

### 结果收集

- [ ] 确认所有 SubAgent 已完成
- [ ] 收集每个 SubAgent 的结果摘要
- [ ] 检查是否有遗漏或未完成的任务

### 报告生成

- [ ] 生成分 Agent 报告（每个 SubAgent 独立章节）
- [ ] 输出整体汇总和关键发现
- [ ] 提供优先级行动项和建议

### 质量验证

- [ ] 报告包含所有关键指标（数量、耗时、完成度）
- [ ] 每个问题有具体位置和修复建议
- [ ] 输出格式符合规范

---

## 性能优化检查

### Token 效率

- [ ] SubAgent 只返回结果摘要，不返回详细推理
- [ ] 主上下文保持干净，只保留决策信息
- [ ] 避免重复加载相同文件

### 并发效率

- [ ] 合理设置并行任务数（不超过 ~10 个）
- [ ] 任务队列避免空闲等待
- [ ] 长时间任务支持断点续执行

---

## 常见问题排查

### SubAgent 未被调用

- [ ] 检查 SubAgent 的 `description` 是否匹配任务
- [ ] 尝试手动调用 `/agents <agent-name>`
- [ ] 确认 SubAgent 配置文件存在

### 执行超时

- [ ] 增加 `maxTurns` 参数
- [ ] 将大任务拆分为更小的子任务
- [ ] 检查是否有死循环或卡住的 SubAgent

### 结果不完整

- [ ] 检查 SubAgent 是否被意外中断
- [ ] 查看 SubAgent 的执行日志
- [ ] 重新执行未完成的子任务

---

## 安全检查

### 权限控制

- [ ] SubAgent 工具权限最小化
- [ ] 敏感操作需要用户确认
- [ ] 检查是否有越权访问

### 数据安全

- [ ] 不泄露敏感信息（API Key、密码等）
- [ ] 不修改未授权的文件
- [ ] 遵守项目的安全规范
